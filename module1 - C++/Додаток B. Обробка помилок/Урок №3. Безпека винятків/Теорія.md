# Урок №3. Безпека винятків

Гарантії безпеки винятків та ідіоми для їх забезпечення.

## Три рівні гарантій

1. **No-throw** (nothrow) — функція ніколи не кидає винятки
2. **Strong** (сильна) — при винятку стан програми не змінюється
3. **Basic** (базова) — при винятку ресурси не витікають, стан коректний

## No-throw гарантія

```c++
void swap(T& a, T& b) noexcept;  // ніколи не кидає
int getValue() const noexcept;    // ніколи не кидає

void process() noexcept
{
    // тільки noexcept операції
    int x = 42;
    int y = x + 1;
}
```

## Strong гарантія (copy-and-swap)

```c++
class BigObject
{
    int* data;
    int size;
    
    void swap(BigObject& other) noexcept
    {
        std::swap(data, other.data);
        std::swap(size, other.size);
    }
    
public:
    void modify(const std::vector<int>& newData)
    {
        // 1. Спочатку створити копію
        BigObject temp(*this);  // може кинути → оригінал не змінений
        
        // 2. Змінити копію
        temp.data = new int[newData.size()];  // може кинути
        // ... копіювання даних ...
        
        // 3. Якщо все успішно — обміняти
        swap(temp);  // noexcept
        // 4. temp знищується зі старими даними
    }
};
```

## Basic гарантія (RAII)

```c++
class FileHandler
{
    FILE* file;
    
public:
    FileHandler(const char* name)
    {
        file = fopen(name, "r");
        if (!file) throw std::runtime_error("Помилка відкриття");
    }
    
    ~FileHandler()
    {
        if (file) fclose(file);  // ресурс завжди звільняється
    }
    
    void readData()
    {
        // навіть якщо виняток — деструктор закриє файл
    }
};
```

## Проблеми без RAII

```c++
// Погано: витік ресурсу при винятку
void badFunction()
{
    int* data = new int[100];
    // ... якщо тут виняток — data витікає!
    delete[] data;
}

// Добре: RAII
void goodFunction()
{
    std::vector<int> data(100);  // деструктор звільнить при винятку
    // ...
}
```

## noexcept оптимізації

```c++
// std::vector використовує noexcept для вибору стратегії:
template<typename T>
void vector<T>::reserve(size_t n)
{
    if (n > capacity())
    {
        T* newBuf = new T[n];
        
        if constexpr (std::is_nothrow_move_constructible_v<T>)
        {
            // переміщення (швидко)
            for (size_t i = 0; i < size(); ++i)
                newBuf[i] = std::move(oldBuf[i]);
        }
        else
        {
            // копіювання (повільно, але безпечно)
            for (size_t i = 0; i < size(); ++i)
                newBuf[i] = oldBuf[i];
        }
    }
}
```

## Приклад: транзакція

```c++
class BankAccount
{
    int balance;
    
public:
    void transfer(BankAccount& to, int amount)
    {
        if (amount > balance)
            throw std::runtime_error("Недостатньо коштів");
        
        // Strong гарантія:
        balance -= amount;
        try
        {
            to.deposit(amount);
        }
        catch (...)
        {
            balance += amount;  // відкат
            throw;
        }
    }
};
```

## Правило

Пишіть код, що не витікає ресурси при винятках. Використовуйте RAII для всіх ресурсів.
