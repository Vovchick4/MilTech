# Урок №41. Пріоритет операцій і правила асоціативності

## Пріоритет операцій

`Пріоритет` визначає, яка операція виконується першою, коли у виразі є кілька різних операторів:

```c++
int x = 5 + 3 * 2;   // * має вищий пріоритет: 5 + (3 * 2) = 11
int y = (5 + 3) * 2; // дужки змінюють порядок: (5 + 3) * 2 = 16
```

## Асоціативність

`Асоціативність` визначає порядок виконання операторів з однаковим пріоритетом:

```c++
// Ліва асоціативність (зліва направо):
int x = 10 - 5 - 2;   // (10 - 5) - 2 = 3

// Права асоціативність (справа напіво):
int a = b = c = 5;    // a = (b = (c = 5)) — всі стають 5
```

## Чому це важливо?

```c++
// Розповсюджена помилка: пріоритет логічних операторів
int x = 5;
if (x & 1 == 0)   // ХІБА! == має вищий пріоритет: x & (1 == 0) → x & 0 → 0
if ((x & 1) == 0) // ПРАВИЛЬНО: перевірка парності через побітове І

// Ще приклад
if (x = 5)        // ХІБА! = має нижчий пріоритет, ніж ==, але це присвоювання!
if (x == 5)       // ПРАВИЛЬНО: порівняння
```

## Правило: використовуйте дужки

Найкраща практика — використовувати дужки для явного вказання порядку, навіть якщо ви знаєте пріоритет:

```c++
// Не покладайтеся на знання пріоритету
int x = a + b * c - d / e;

// Пишіть зрозуміло
int x = a + (b * c) - (d / e);

// Складні умови
if ((a > 0 && b > 0) || (c == 0))
```

## Приклад

```c++
#include <iostream>

int main()
{
    // Без дужок — покладаємося на пріоритет
    int result1 = 5 + 3 * 4 - 8 / 2;  // 5 + 12 - 4 = 13
    
    // З дужками — читається краще
    int result2 = 5 + (3 * 4) - (8 / 2);  // 13
    
    std::cout << result1 << " " << result2;
    return 0;
}
```
