# Урок №103. Передача по значенню

При передачі за значенням функція отримує КОПІЮ аргументу.

## Як це працює

```c++
void func(int x)  // x — копія аргументу
{
    x = 100;  // зміна копії
}

int main()
{
    int a = 5;
    func(a);     // a копіюється в x
    std::cout << a;  // 5 — оригінал не змінився
    return 0;
}
```

## Переваги

```c++
// 1. Оригінал захищений від змін
void process(int x) { x = 0; }
int value = 42;
process(value);  // value все ще 42

// 2. Простота — не потрібно турбуватися про час життя
void print(std::string s) { std::cout << s; }
// s можна змінювати всередині
```

## Недолік: копіювання великих об'єктів

```c++
struct BigData { int data[1000]; };

void process(BigData d)  // копіює 4000 байтів! (повільно)
{
    // використання d
}

// Краще:
void process(const BigData& d)  // без копіювання
{
    // використання d
}
```

## Коли використовувати?

```c++
// Добре для малих типів:
void func(int x);       // 4 байти — копіювати дешево
void func(double x);    // 8 байтів — дешево
void func(char c);      // 1 байт — дешево
void func(bool b);      // дешево

// Погано для великих типів:
void func(std::string s);       // копіює рядок (дорого)
void func(std::vector<int> v);  // копіює весь вектор (дуже дорого)
```

## Семантика

```c++
void byValue(int x)  // копія
{
    x = x + 1;
}

int main()
{
    int original = 5;
    byValue(original);
    std::cout << original;  // 5 — без змін
    return 0;
}
```

## Таблиця

| Спосіб | Копіювання | Зміна оригіналу | Типове використання |
|--------|-----------|-----------------|-------------------|
| За значенням | Так | Ні | Малі типи (int, char, bool) |
| За посиланням | Ні | Так | Зміна аргументу |
| const посиланням | Ні | Ні | Великі об'єкти (читання) |

## Правило

Малі типи (до 8-16 байтів) передавайте за значенням. Великі об'єкти — за const посиланням.

## Тест

1. Чи може функція змінити оригінал при передачі за значенням?
2. Чому передача великої структури за значенням — погано?
3. Які типи варто передавати за значенням?
