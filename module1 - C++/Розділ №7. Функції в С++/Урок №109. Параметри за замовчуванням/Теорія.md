# Урок №109. Параметри за замовчуванням

Параметри функції можуть мати значення за замовчуванням.

## Базовий синтаксис

```c++
void greet(std::string name = "світ")
{
    std::cout << "Привіт, " << name << "!\n";
}

greet();          // "Привіт, світ!"
greet("Анна");    // "Привіт, Анна!"
```

## Кілька параметрів

Параметри за замовчуванням мають бути після звичайних:

```c++
void printInfo(std::string name, int age = 0, std::string country = "Україна");
// OK

// void printInfo(std::string name = "Анонім", int age);  // ПОМИЛКА!
```

## Виклик

```c++
void show(int a = 1, int b = 2, int c = 3);

show();              // a=1, b=2, c=3
show(10);            // a=10, b=2, c=3
show(10, 20);        // a=10, b=20, c=3
show(10, 20, 30);    // a=10, b=20, c=30
// show(10, , 30);   // ПОМИЛКА! не можна пропустити
```

## Прототип vs Визначення

Значення за замовчуванням пишуться ТІЛЬКИ в прототипі:

```c++
// header.h
void print(int x = 5);  // ТУТ

// source.cpp
#include "header.h"
void print(int x)        // НЕ ТУТ
{
    for (int i = 0; i < x; ++i)
        std::cout << "#";
}
```

## Зміна значень

```c++
void configure(int speed = 5, int volume = 10);

configure();            // 5, 10
configure(15);          // 15, 10
configure(15, 20);      // 15, 20
```

## Перевантаження + за замовчуванням

```c++
// Неоднозначність:
void print(int x);
void print(int x, int y = 10);

print(5);  // ПОМИЛКА! яку функцію викликати?
```

## Правила

1. Значення за замовчуванням — справа
2. Тільки в прототипі (не у визначенні)
3. Не створюйте неоднозначності з перевантаженими функціями

## Правило

Використовуйте параметри за замовчуванням для додаткових/необов'язкових параметрів, щоб спростити API.

## Тест

1. Чи можна мати звичайні параметри після параметрів за замовчуванням?
2. Де пишуться значення за замовчуванням: у прототипі чи визначенні?
3. Чи можна викликати `func(10, , 30)`?
