# Урок №104. Передача по посиланню

При передачі за посиланням функція отримує посилання на оригінал, без копіювання.

## Синтаксис

```c++
void func(int& x)  // x — посилання
{
    x = 100;  // змінює оригінал
}
```

## Зміна аргументу

```c++
void swap(int& a, int& b)
{
    int temp = a;
    a = b;
    b = temp;
}

int x = 5, y = 10;
swap(x, y);
std::cout << x << " " << y;  // 10 5
```

## Економія копіювання

```c++
#include <iostream>
#include <vector>

void printSize(const std::vector<int>& vec)  // const — без копії, без зміни
{
    std::cout << "Розмір: " << vec.size() << "\n";
}

int main()
{
    std::vector<int> data(1000000);  // великий вектор
    printSize(data);  // без копіювання!
    return 0;
}
```

## Повернення через параметр

```c++
void getDimensions(int& width, int& height)
{
    width = 1920;
    height = 1080;
}

int w, h;
getDimensions(w, h);
std::cout << w << "x" << h;  // 1920x1080
```

## const T& — читання без копії

```c++
void print(const std::string& s)
{
    std::cout << s;  // тільки читання
    // s[0] = 'X';  // ПОМИЛКА! const
}
```

## Не можна прив'язати тимчасовий об'єкт до звичайного посилання

```c++
void func(int& x) {}

func(5);  // ПОМИЛКА! 5 — тимчасовий (rvalue)

void func(const int& x) {}
func(5);  // OK: const T& приймає тимчасові
```

## Коли використовувати?

```c++
// Потрібно змінити → T&
void increment(int& x);

// Великий об'єкт, не змінюємо → const T&
void print(const std::string& s);

// Малий тип, не змінюємо → T (за значенням)
bool isEven(int x);
```

## Правило

Використовуйте посилання для зміни аргументів і для уникнення копіювання великих об'єктів.

## Тест

1. Як передати великий об'єкт без копіювання і без можливості зміни?
2. Чи можна передати літерал `5` у функцію з `int&`?
3. Як повернути кілька значень з функції?
