# Урок №106. Повернення значень

Функція може повертати значення різними способами.

## Повернення за значенням

```c++
int add(int a, int b)
{
    return a + b;  // повертає копію результату
}

int result = add(3, 4);  // 7
```

Для малих типів — оптимально.

## Повернення за посиланням

```c++
int& getElement(int arr[], int index)
{
    return arr[index];  // повертає посилання
}

int data[5] = {1, 2, 3, 4, 5};
getElement(data, 2) = 10;  // data[2] = 10
std::cout << data[2];  // 10
```

## Повернення константного посилання

```c++
class Storage
{
    std::vector<int> m_data;
public:
    const std::vector<int>& getData() const { return m_data; }
};
```

## Ніколи не повертайте посилання на локальну змінну!

```c++
int& bad()  // НЕБЕЗПЕЧНО!
{
    int x = 5;
    return x;  // x знищиться після виходу!
}

int& ref = bad();
std::cout << ref;  // НЕВИЗНАЧЕНО!
```

## Повернення за адресою

```c++
int* findValue(int arr[], int size, int target)
{
    for (int i = 0; i < size; ++i)
        if (arr[i] == target)
            return &arr[i];  // адреса існуючого об'єкта
    return nullptr;  // не знайдено
}
```

## RVO та NRVO (оптимізація повернення)

Компілятор часто оптимізує копіювання при поверненні:

```c++
std::vector<int> createData()
{
    std::vector<int> result = {1, 2, 3, 4, 5};
    return result;  // RVO: копіювання може бути оптимізовано
}
```

## auto як тип повернення (C++14)

```c++
auto add(int a, int b)
{
    return a + b;  // auto → int
}

auto createPair()
{
    return std::make_pair(1, "hello");
}
```

## Таблиця

| Спосіб | Копія | Можна змінити | nullptr | Локальні змінні |
|--------|-------|--------------|---------|----------------|
| За значенням | Так | Ні | Н/Д | Так |
| За посиланням | Ні | Так | Ні | Ні |
| const посиланням | Ні | Ні | Ні | Ні |
| За адресою | Ні | Так | Так | Ні |

## Правило

За замовчуванням повертайте за значенням. Посилання/вказівники повертайте тільки для існуючих об'єктів (не локальних).
