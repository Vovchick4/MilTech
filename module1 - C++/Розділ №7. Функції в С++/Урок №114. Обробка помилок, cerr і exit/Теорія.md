# Урок №114. Обробка помилок, cerr і exit()

## std::cerr — потік помилок

`std::cerr` — потік виведення для помилок (на відміну від `std::cout`):

```c++
#include <iostream>

int main()
{
    int x;
    std::cin >> x;
    
    if (std::cin.fail())
    {
        std::cerr << "Помилка вводу!\n";  // вивід у stderr
        return 1;  // код помилки
    }
    
    std::cout << "Ви ввели: " << x << "\n";  // вивід у stdout
    return 0;
}
```

## std::exit()

`std::exit()` негайно завершує програму:

```c++
#include <cstdlib>

void criticalError()
{
    std::cerr << "Критична помилка!\n";
    std::exit(1);  // завершення програми
    // код після exit() не виконується
}
```

## Коди завершення

```c++
return 0;       // успіх
return 1;       // загальна помилка
return -1;      // помилка
std::exit(EXIT_SUCCESS);  // 0
std::exit(EXIT_FAILURE);  // 1
```

## Перевірка результату функції

```c++
#include <iostream>

bool divide(int a, int b, int& result)
{
    if (b == 0)
    {
        std::cerr << "Помилка: ділення на 0\n";
        return false;
    }
    result = a / b;
    return true;
}

int main()
{
    int result;
    if (divide(10, 0, result))
        std::cout << result;
    else
        std::cerr << "Не вдалося обчислити\n";
    
    return 0;
}
```

## Альтернативи: std::optional

```c++
#include <optional>

std::optional<int> divide(int a, int b)
{
    if (b == 0)
        return std::nullopt;  // помилка
    return a / b;
}

int main()
{
    auto result = divide(10, 2);
    if (result)
        std::cout << *result;  // 5
    return 0;
}
```

## cerr vs cout

```c++
std::cout << "Звичайний вивід\n";  // stdout — буферизований
std::cerr << "Помилка\n";          // stderr — небуферизований
```

## Правило

Використовуйте `std::cerr` для помилок і `std::cout` для звичайного виводу. Це дозволяє перенаправляти їх окремо:

```bash
program > output.txt 2> errors.txt
```
