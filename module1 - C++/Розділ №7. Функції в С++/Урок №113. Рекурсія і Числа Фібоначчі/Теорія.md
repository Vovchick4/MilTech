# Урок №113. Рекурсія і Числа Фібоначчі

Рекурсія — коли функція викликає саму себе.

## Базовий випадок та рекурсивний випадок

```c++
void countdown(int n)
{
    if (n <= 0)          // базовий випадок
        return;
    
    std::cout << n << "\n";
    countdown(n - 1);    // рекурсивний виклик
}
```

## Факторіал

```c++
int factorial(int n)
{
    if (n <= 1) return 1;         // базовий випадок
    return n * factorial(n - 1);  // рекурсивний
}

// factorial(5) = 5 * 4 * 3 * 2 * 1 = 120
```

## Числа Фібоначчі

Послідовність: 0, 1, 1, 2, 3, 5, 8, 13, 21...
- F(0) = 0
- F(1) = 1
- F(n) = F(n-1) + F(n-2)

```c++
int fib(int n)
{
    if (n <= 0) return 0;         // базовий
    if (n == 1) return 1;         // базовий
    return fib(n - 1) + fib(n - 2); // рекурсивний
}
```

## Проблема: експоненційна складність

```c++
fib(5) → fib(4) + fib(3)
         → fib(3) + fib(2) + fib(2) + fib(1)
         → ...
// Кількість викликів зростає як O(2ⁿ)!
```

## Ітеративне рішення (краще)

```c++
int fibIterative(int n)
{
    if (n <= 0) return 0;
    if (n == 1) return 1;
    
    int prev = 0, curr = 1;
    for (int i = 2; i <= n; ++i)
    {
        int next = prev + curr;
        prev = curr;
        curr = next;
    }
    return curr;
}
```

## Стек викликів при рекурсії

```c++
factorial(3):
  → factorial(3)
    → factorial(2)
      → factorial(1)
        → return 1
      → return 2 * 1 = 2
    → return 3 * 2 = 6
```

## Переваги та недоліки

| Переваги | Недоліки |
|----------|----------|
| Код часто простіший | Витрати на виклики |
| Природно для деяких задач | Може призвести до stack overflow |
| Дерева, графи, обходи | Не завжди ефективно |

## Коли використовувати?

```c++
// Добре для рекурсії: дерева, обходи, розділяй-і-володарюй
void traverse(TreeNode* node)
{
    if (!node) return;
    traverse(node->left);
    std::cout << node->value;
    traverse(node->right);
}
```

## Правило

Рекурсія — потужний інструмент, але зважайте на глибину викликів. Для простих задач (факторіал, Фібоначчі) ітерація часто краща.
