# Урок №108. Перевантаження функцій

`Перевантаження` (overloading) — кілька функцій з ОДНАКОВИМ ім'ям, але РІЗНИМИ параметрами.

## Правила

Функції вважаються різними, якщо відрізняються:
1. Кількість параметрів
2. Типи параметрів
3. Порядок параметрів

## Приклад

```c++
#include <iostream>

int max(int a, int b)
{
    return (a > b) ? a : b;
}

double max(double a, double b)
{
    return (a > b) ? a : b;
}

int max(int a, int b, int c)
{
    return max(max(a, b), c);
}

int main()
{
    std::cout << max(5, 10) << "\n";       // викликає int версію
    std::cout << max(3.14, 2.71) << "\n";  // викликає double версію
    std::cout << max(1, 5, 3) << "\n";     // викликає 3-параметрову
    return 0;
}
```

## Що НЕ можна перевантажувати?

```c++
// Тільки тип повернення — НЕДОСТАТНЬО:
int getValue();
double getValue();  // ПОМИЛКА!

// Імена параметрів — неважливі:
void func(int x);
void func(int y);   // ПОМИЛКА! те саме
```

## Перевантаження та const

```c++
void print(int x);
void print(const int x);  // ПОМИЛКА! те саме для caller

// Але для посилань/вказівників — працює:
void process(int& x);
void process(const int& x);  // OK — різні типи
```

## Перевантаження конструкторів

```c++
class Point
{
    int x, y;
public:
    Point() : x{0}, y{0} {}           // без параметрів
    Point(int val) : x{val}, y{val} {} // один параметр
    Point(int x, int y) : x{x}, y{y} {} // два параметри
};
```

## Неоднозначність

```c++
void print(int x);
void print(double y);

print(5);     // OK → int
print(3.14);  // OK → double
// print('A');  // неоднозначно? char → int або char → double?
```

## Користь

```c++
// Замість різних імен:
int maxInt(int a, int b);
double maxDouble(double a, double b);

// Одне ім'я:
int max(int a, int b);
double max(double a, double b);
// Інтуїтивно зрозуміло
```
