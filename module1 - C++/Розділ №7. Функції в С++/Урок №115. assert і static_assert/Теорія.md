# Урок №115. assert і static_assert

## assert (runtime)

`assert` — макрос для перевірки умов під час виконання:

```c++
#include <cassert>

int divide(int a, int b)
{
    assert(b != 0);  // якщо b == 0 — програма завершується
    return a / b;
}
```

## Як це працює

```c++
assert(умова);
// Якщо умова false: виводить повідомлення і завершує програму
```

## Використання

```c++
#include <cassert>
#include <iostream>

double sqrt(double x)
{
    assert(x >= 0.0);  // перевірка передумови
    // обчислення...
    return 0.0;
}

void processArray(const int* arr, int size)
{
    assert(arr != nullptr);  // не нульовий вказівник
    assert(size > 0);        // додатній розмір
    
    for (int i = 0; i < size; ++i)
        ; // обробка
}
```

## Відключення assert

```c++
#define NDEBUG  // перед #include <cassert>
#include <cassert>

assert(false);  // ігнорується! (NDEBUG визначено)
```

У release-збірках `NDEBUG` зазвичай визначено, тому assert не працює.

## static_assert (compile-time)

`static_assert` — перевірка на етапі компіляції:

```c++
static_assert(умова, "повідомлення");
```

```c++
#include <iostream>

static_assert(sizeof(int) == 4, "int має бути 4 байти!");
static_assert(CHAR_BIT == 8, "Байт має бути 8 бітів!");

template<typename T>
void checkType()
{
    static_assert(std::is_integral_v<T>, "T має бути цілим типом!");
}

int main()
{
    checkType<int>();    // OK
    // checkType<double>();  // ПОМИЛКА КОМПІЛЯЦІЇ
    return 0;
}
```

## assert vs static_assert

```c++
assert(x > 0);          // runtime — перевіряє значення
static_assert(N > 0);   // compile-time — перевіряє константу
```

## Кращі практики

```c++
void process(int* data, int size)
{
    // Перевірки інваріантів
    assert(data != nullptr);
    assert(size >= 0);
    
    // ...
    
    // Перевірка постумов
    assert(result != nullptr);
}
```

## Правило

Використовуйте `assert` для відлагодження (перевірка інваріантів, передумов). Використовуйте `static_assert` для перевірки констант і типів на етапі компіляції.
