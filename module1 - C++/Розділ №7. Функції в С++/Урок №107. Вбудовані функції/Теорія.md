# Урок №107. Вбудовані функції

`inline` — підказка компілятору вбудувати код функції безпосередньо в місце виклику.

## Навіщо?

Виклик функції має накладні витрати (збереження регістрів, перехід). Для маленьких функцій це може бути дорожче, ніж сам код.

## Синтаксис

```c++
inline int max(int a, int b)
{
    return (a > b) ? a : b;
}
```

## Як це працює

```c++
// Без inline — виклик функції:
int result = max(5, 10);
// → зберегти регістри, перейти, виконати, повернутися

// З inline — код підставляється:
int result = (5 > 10) ? 5 : 10;
// → без виклику, без накладних витрат
```

## inline — це РЕКОМЕНДАЦІЯ

Компілятор може проігнорувати `inline`:

```c++
inline void largeFunction()  // компілятор проігнорує
{
    // багато коду...
}
```

## constexpr-функції

constexpr-функції ТАКОЖ неявно inline:

```c++
constexpr int square(int x) { return x * x; }
// неявно inline — може обчислюватися на етапі компіляції
```

## inline та заголовкові файли

Функції, визначені в заголовкових файлах, мають бути inline:

```c++
// math.h
#pragma once

inline int add(int a, int b)  // inline — щоб не було "multiple definition"
{
    return a + b;
}

// Або constexpr:
constexpr int multiply(int a, int b) { return a * b; }
```

## Сучасна практика

У сучасному C++ `inline` використовується переважно для:
- Функцій у заголовкових файлах
- `constexpr` функцій (неявно inline з C++17)

## Правило

Покладайтеся на компілятор — він краще знає, що вбудовувати. `inline` використовуйте тільки для заголовкових файлів.
