# Урок №58. Неявна конвертація типів даних

`Неявна конвертація` (або приведення) — автоматичне перетворення одного типу даних в інший компілятором.

## Коли відбувається?

### 1. В арифметичних операціях з різними типами

```c++
int a = 5;
double b = 2.5;
double c = a + b;  // a неявно конвертується в double: 5.0 + 2.5 = 7.5
```

### 2. При присвоюванні

```c++
int x = 3.14;    // double → int: x = 3 (втрата даних!)
double y = 5;    // int → double: y = 5.0
```

### 3. При передачі аргументів функції

```c++
void func(double x) { }

int main()
{
    func(5);  // int → double: func(5.0)
    return 0;
}
```

### 4. В умовних виразах

```c++
if (5)        // int → bool: 5 → true
if (0)        // int → bool: 0 → false
```

## Правила конвертації (ієрархія)

```c++
long double (найвищий)
    ↑
double
    ↑
float
    ↑
unsigned long long
    ↑
long long
    ↑
unsigned long
    ↑
long
    ↑
unsigned int
    ↑
int
    ↑
short / char (найнижчий)
```

У виразі з різними типами, нижчий тип піднімається до вищого:

```c++
int + double → double
short + int → int
float + double → double
```

## Звужувальні конвертації (narrowing)

Коли тип втрачає дані:

```c++
int x = 3.14;      // 3 (втрата дробової частини)
short y = 100000;  // переповнення (31072 або інше значення)
char c = 300;      // переповнення
```

Uniform-ініціалізація `{}` блокує звуження:

```c++
int x { 3.14 };    // ПОМИЛКА КОМПІЛЯЦІЇ!
int y = 3.14;      // OK (але небезпечно)
```

## Числове просування (integral promotion)

Менші цілі типи автоматично "просуваються" до `int`:

```c++
short a = 5;
short b = 10;
auto c = a + b;   // c — int, не short!
```

```c++
char ch = 'A';
std::cout << ch;              // 'A'
std::cout << ch + 1;          // 66 (char просунувся до int)
std::cout << static_cast<char>(ch + 1);  // 'B'
```

## Небезпечні моменти

```c++
unsigned int u = 5;
int s = -10;

if (u + s > 0)  // s перетворюється на unsigned: -10 → 4294967286
{
    // цей код виконається! (може бути несподівано)
}
```

## Таблиця конвертацій

| З | В | Результат |
|---|----|-----------|
| int → double | 5 → 5.0 | Безпечно |
| double → int | 3.7 → 3 | Втрата дробової частини |
| int → bool | 0 → false, інше → true | Безпечно |
| unsigned → signed | -1 → велике число | Небезпечно |
| float → int | 1.9 → 1 | Втрата точності |

## Правило

Уникайте змішування signed та unsigned типів. Для явної конвертації з перевіркою використовуйте `{}` uniform-ініціалізацію.

## Тест

1. Що буде: `double x = 5 + 0.5;`?
2. Чому `int x { 3.14 };` — помилка?
3. Як `char` просувається в арифметичних виразах?
