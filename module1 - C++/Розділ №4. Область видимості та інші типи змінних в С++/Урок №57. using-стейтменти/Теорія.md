# Урок №57. using-стейтменти

`using` дозволяє використовувати імена з простору імен без префікса.

## using declaration

`Using declaration` — імпортує КОНКРЕТНЕ ім'я:

```c++
using std::cout;   // тепер можна писати просто cout
using std::endl;

int main()
{
    cout << "Hello" << endl;  // без std::
    return 0;
}
```

## using directive (using namespace)

`Using directive` — імпортує ВСІ імена з простору імен:

```c++
using namespace std;  // ВСІ імена з std

int main()
{
    cout << "Hello";  // OK
    string s = "Hi";  // OK
    vector<int> v;    // OK
    return 0;
}
```

## Чому `using namespace std;` — погано?

1. **Конфлікти імен** — ваша змінна може збігтися з ім'ям зі std:

```c++
using namespace std;

int count = 5;  // конфлікт! std::count існує в <algorithm>
string name;    // можливий конфлікт у майбутньому
```

2. **Неоднозначність** — який саме `count` ви маєте на увазі?

3. **Забруднення в заголовкових файлах** — поширюється на всі файли, що включають ваш header

## Коли використовувати?

### Кращий підхід: явний std::

```c++
std::cout << "Hello";
std::string name;
std::vector<int> data;
```

### Прийнятно: using declaration в обмеженій області

```c++
int main()
{
    using std::cout;  // тільки для main()
    cout << "Hello";
    return 0;
}
```

### Ніколи: using namespace std у заголовкових файлах

```c++
// header.h
using namespace std;  // НІКОЛИ!
```

## Ієрархія

```c++
int g_x { 5 };        // глобальна змінна

namespace Foo
{
    int g_x { 10 };   // Foo::g_x
    
    namespace Bar
    {
        int g_x { 15 };  // Foo::Bar::g_x
    }
}

using namespace Foo;
using namespace Bar;      // Foo::Bar

int main()
{
    std::cout << g_x;     // ПОМИЛКА: неоднозначність!
    std::cout << ::g_x;   // 5 (глобальна)
    std::cout << Foo::g_x;  // 10
    return 0;
}
```

## Правило

Найкраще: завжди використовуйте `std::` явно. Якщо дуже хочеться using — обмежуйте його найменшою областю видимості.
