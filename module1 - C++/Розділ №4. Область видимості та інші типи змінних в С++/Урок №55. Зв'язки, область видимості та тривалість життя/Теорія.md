# Урок №55. Зв'язки, область видимості та тривалість життя

## Три концепції

| Концепція | Опис |
|-----------|------|
| **Область видимості (scope)** | Де змінна доступна за ім'ям |
| **Тривалість життя (lifetime)** | Як довго змінна існує в пам'яті |
| **Зв'язок (linkage)** | Чи доступна змінна в інших файлах |

## Зв'язок (Linkage)

`Зв'язок` визначає, чи може змінна бути використана в інших файлах:

| Тип зв'язку | Опис |
|-------------|------|
| **No linkage** | Доступна тільки в поточній функції/блоці |
| **Internal linkage** | Доступна в поточному файлі |
| **External linkage** | Доступна в інших файлах програми |

### No linkage

Локальні змінні та параметри:

```c++
void func()
{
    int x { 5 };     // no linkage — доступна тільки тут
}
```

### Internal linkage

Змінна видима в межах одного файлу:

```c++
// Способи:
static int g_fileVar { 5 };

namespace {           // анонімний namespace
    int g_fileVar2 { 10 };
}

const int g_const { 15 };  // const змінні мають internal linkage
```

### External linkage

Видима у всій програмі:

```c++
int g_global { 5 };   // external linkage (за замовчуванням)

extern int g_extern;  // оголошення extern — посилання на змінну в іншому файлі
```

## Таблиця за типами змінних

| Тип змінної | Область видимості | Тривалість життя | Зв'язок |
|-------------|-------------------|-------------------|---------|
| Локальна | Блок {} | Блок {} | No linkage |
| static локальна | Блок {} | Вся програма | No linkage |
| Глобальна (не const) | Весь файл | Вся програма | External |
| static глобальна | Весь файл | Вся програма | Internal |
| const глобальна | Весь файл | Вся програма | Internal |
| extern глобальна | Весь файл | Вся програма | External |

## Приклад з кількома файлами

**a.cpp:**
```c++
int g_external { 5 };       // external linkage
static int g_internal { 10 }; // internal linkage

void func()
{
    static int s_local { 0 };  // no linkage, lifetime = вся програма
    ++s_local;
}
```

**b.cpp:**
```c++
extern int g_external;    // посилання на змінну з a.cpp
// extern int g_internal; // ПОМИЛКА! internal linkage
```

## Правило

Використовуйте:
- Internal linkage (анонімний namespace) для файлових змінних
- External linkage тільки коли дійсно потрібен доступ з інших файлів
- static локальні змінні для збереження стану в функції
