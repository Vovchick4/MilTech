# Урок №53. Чому глобальні змінні – це зло?

Хоча глобальні змінні зручні, вони створюють багато проблем.

## 1. Непередбачувані зміни

Будь-яка функція може змінити глобальну змінну:

```c++
int g_score { 100 };

void someFunction()
{
    g_score = 0;  // десь "випадково" змінили
}

int main()
{
    someFunction();
    std::cout << g_score;  // 0 — а має бути 100!
    return 0;
}
```

## 2. Складність налагодження

Коли глобальна змінна має неправильне значення, потрібно перевірити ВСІ місця, де вона змінюється:

```c++
void funcA() { /* може змінити g_state */ }
void funcB() { /* може змінити g_state */ }
void funcC() { /* може змінити g_state */ }
// яка з них встановила неправильне значення?
```

## 3. Залежність між функціями

Функції стають залежними від глобального стану:

```c++
int g_mode { 1 };

void draw()
{
    if (g_mode == 1) /* ... */ 
    else if (g_mode == 2) /* ... */
}
// draw() не можна використовувати без глобальної змінної!
```

## 4. Проблеми з багатопотоковістю

Кілька потоків, що одночасно змінюють глобальну змінну — рецепт катастрофи.

## 5. Ініціалізація

Порядок ініціалізації глобальних змінних у різних файлах не визначений:

```c++
// a.cpp
int g_a { func() };  // func() може залежати від g_b

// b.cpp
int g_b { 5 };       // чи буде g_b ініціалізовано до g_a? Невідомо!
```

## Коли глобальні змінні виправдані?

1. **Глобальні константи** — `constexpr` значення (PI, MAX_SIZE)
2. **Логування** — глобальний об'єкт логера
3. **Конфігурація** — параметри програми (але краще через параметри)
4. **Інтерфейс з C-бібліотеками**

## Альтернативи глобальним змінним

```c++
// Погано: глобальна змінна
int g_mode { 1 };
void func() { if (g_mode == 1) ... }

// Добре: параметр функції
void func(int mode) { if (mode == 1) ... }

// Ще краще: клас з інкапсуляцією
class Config {
    int m_mode { 1 };
public:
    void setMode(int m) { m_mode = m; }
    int getMode() const { return m_mode; }
};
```

## Правило

Уникайте глобальних змінних. Якщо без них не обійтися — робіть їх `const` або `constexpr`. Використовуйте префікс `g_`.
