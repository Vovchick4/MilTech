# Урок №1. Основи CMake

CMake — система збірки (build system generator), що створює Makefile, проекти Visual Studio тощо.

## Мінімальний CMakeLists.txt

```cmake
cmake_minimum_required(VERSION 3.10)

project(MyProject)

add_executable(myapp main.cpp)
```

## Збірка

```bash
# 1. Створити директорію збірки
mkdir build
cd build

# 2. Згенерувати файли збірки
cmake ..

# 3. Зібрати
cmake --build .

# Або на Windows з Visual Studio:
cmake .. -G "Visual Studio 17 2022"
cmake --build . --config Release
```

## Декілька файлів

```
project/
├── CMakeLists.txt
├── main.cpp
├── math.cpp
└── math.h
```

```cmake
cmake_minimum_required(VERSION 3.10)
project(MathApp)

add_executable(mathapp main.cpp math.cpp math.h)
```

## Додавання бібліотеки

```cmake
# Створити бібліотеку
add_library(mylib STATIC lib.cpp lib.h)

# Під'єднати до виконуваного файлу
add_executable(myapp main.cpp)
target_link_libraries(myapp PRIVATE mylib)
```

## Заголовочні файли

```cmake
# Вказати директорію з заголовками
target_include_directories(myapp PRIVATE ${PROJECT_SOURCE_DIR}/include)
```

## Стандарт C++

```cmake
# Встановити стандарт C++
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# Або для конкретної цілі:
target_compile_features(myapp PRIVATE cxx_std_17)
```

## Приклад: повний проект

```cmake
cmake_minimum_required(VERSION 3.10)
project(Calculator)

# Стандарт C++
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# Виконуваний файл
add_executable(calc
    main.cpp
    calculator.cpp
    calculator.h
)

# Додаткові налаштування
target_include_directories(calc PRIVATE ${CMAKE_CURRENT_SOURCE_DIR})
```

## Команди

```bash
# Release збірка
cmake .. -DCMAKE_BUILD_TYPE=Release
cmake --build .

# Debug збірка
cmake .. -DCMAKE_BUILD_TYPE=Debug
cmake --build .

# Очистка
cmake --build . --target clean
```

## Правило

Завжди збирайте в окремій директорії (build/). Ніколи не запускайте cmake в директорії з вихідним кодом.
