# Урок №2. Бібліотеки в CMake

Статичні, динамічні бібліотеки та зовнішні залежності.

## Типи бібліотек

```cmake
# Статична (.lib / .a)
add_library(mylib STATIC source.cpp)

# Динамічна (.dll / .so)
add_library(mylib SHARED source.cpp)

# Об'єктна (без лінкування)
add_library(mylib OBJECT source.cpp)

# Інтерфейсна (тільки заголовки)
add_library(mylib INTERFACE)
```

## Структура проекту

```
project/
├── CMakeLists.txt
├── src/
│   ├── main.cpp
├── libs/
│   ├── math/
│   │   ├── CMakeLists.txt
│   │   ├── math.cpp
│   │   └── math.h
```

```cmake
# Головний CMakeLists.txt
cmake_minimum_required(VERSION 3.10)
project(MyProject)

add_subdirectory(libs/math)

add_executable(myapp src/main.cpp)
target_link_libraries(myapp PRIVATE mathlib)
```

```cmake
# libs/math/CMakeLists.txt
add_library(mathlib STATIC math.cpp math.h)
target_include_directories(mathlib PUBLIC ${CMAKE_CURRENT_SOURCE_DIR})
```

## Зовнішні бібліотеки (find_package)

```cmake
# Знайти встановлену бібліотеку
find_package(Boost REQUIRED COMPONENTS filesystem system)

target_link_libraries(myapp PRIVATE Boost::filesystem Boost::system)

# Знайти OpenCV
find_package(OpenCV REQUIRED)
target_link_libraries(myapp PRIVATE ${OpenCV_LIBS})
```

## FetchContent (C++17+)

```cmake
include(FetchContent)

FetchContent_Declare(
    nlohmann_json
    GIT_REPOSITORY https://github.com/nlohmann/json.git
    GIT_TAG v3.10.5
)

FetchContent_MakeAvailable(nlohmann_json)

target_link_libraries(myapp PRIVATE nlohmann_json::nlohmann_json)
```

## Інтерфейсна бібліотека

```cmake
# Для заголовочних бібліотек (header-only)
add_library(myheaders INTERFACE)
target_include_directories(myheaders INTERFACE include/)
target_compile_features(myheaders INTERFACE cxx_std_17)
```

## Приклад: статична бібліотека

```cmake
# CMakeLists.txt
cmake_minimum_required(VERSION 3.10)
project(MathLib)

add_library(mathlib STATIC src/math.cpp)
target_include_directories(mathlib PUBLIC include)
```

## Правило

Статичні бібліотеки простіші в розповсюдженні. Динамічні — менший розмір, але потребують DLL/SO.
