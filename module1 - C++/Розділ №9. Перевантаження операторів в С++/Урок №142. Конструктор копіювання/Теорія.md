# Урок №142. Конструктор копіювання

Конструктор копіювання створює новий об'єкт як копію існуючого. Якщо клас має динамічні ресурси — потрібен власний.

## Синтаксис

```c++
class MyClass
{
public:
    MyClass(const MyClass& other);  // конструктор копіювання
};
```

## Коли викликається?

```c++
class Point
{
public:
    Point(int x, int y) { /* ... */ }
    Point(const Point& other) { std::cout << "Copy\n"; }
};

int main()
{
    Point p1(1, 2);
    Point p2 = p1;    // Копіювання (ініціалізація)
    Point p3(p1);     // Копіювання (пряма ініціалізація)
    
    std::vector<Point> vec;
    vec.push_back(p1);  // Копіювання в вектор
    
    return p2;  // може бути копіювання
}
```

## Глибоке копіювання

```c++
class DynamicArray
{
    int* data;
    int size;
    
public:
    DynamicArray(int s) : size{s}
    {
        data = new int[size]{};  // ініціалізація нулями
    }
    
    // Глибоке копіювання:
    DynamicArray(const DynamicArray& other) : size{other.size}
    {
        data = new int[size];
        for (int i = 0; i < size; ++i)
            data[i] = other.data[i];
    }
    
    ~DynamicArray() { delete[] data; }
    
    void set(int index, int value) { data[index] = value; }
    int get(int index) const { return data[index]; }
};
```

## Проблема поверхневого копіювання

```c++
class ShallowCopy
{
    int* data;
public:
    ShallowCopy(int v) { data = new int{v}; }
    // НЕМАЄ конструктора копіювання — компілятор зробить поверхневе
    
    ~ShallowCopy() { delete data; }
};

int main()
{
    ShallowCopy a(42);
    ShallowCopy b = a;  // поверхневе: обидва вказують на ту саму пам'ять
    // Деструктор b видалить data, деструктор a — подвійне звільнення!
    return 0;
}
```

## Копіювання в спадкуванні

```c++
class Base
{
    int baseField;
public:
    Base(int v) : baseField{v} {}
    Base(const Base& other) : baseField{other.baseField} {}
};

class Derived : public Base
{
    int derivedField;
public:
    Derived(int a, int b) : Base{a}, derivedField{b} {}
    
    Derived(const Derived& other)
        : Base{other}  // виклик конструктора копіювання Base
        , derivedField{other.derivedField}
    {}
};
```

## Заборона копіювання

```c++
class NonCopyable
{
public:
    NonCopyable(const NonCopyable&) = delete;  // заборона копіювання
    NonCopyable& operator=(const NonCopyable&) = delete;
};
```

## Копіювання через ініціалізацію списком

```c++
class Person
{
    std::string name;
    int age;
    
public:
    Person(const std::string& n, int a) : name{n}, age{a} {}
    // Компілятор генерує конструктор копіювання (для std::string — правильний)
};
```

## Правило

Якщо клас керує ресурсами (new/delete, файли, сокети), реалізуйте глибоке копіювання. Інакше — автоматично згенерованого достатньо.
