# Урок №150. Перевантаження new і delete

Можна визначити власні версії `new` і `delete` для класу або глобально.

## Навіщо?

- Відстеження витоків пам'яті
- Використання пулу пам'яті
- Профілювання
- Вирівнювання

## Локальне (для одного класу)

```c++
class Tracked
{
    int data;
    
public:
    static void* operator new(size_t size)
    {
        std::cout << "Виділено " << size << " байт\n";
        void* ptr = std::malloc(size);
        return ptr;
    }
    
    static void operator delete(void* ptr)
    {
        std::cout << "Звільнено\n";
        std::free(ptr);
    }
};

int main()
{
    Tracked* obj = new Tracked();  // "Виділено 4 байт"
    delete obj;                     // "Звільнено"
    return 0;
}
```

## Для масивів

```c++
class Tracked
{
public:
    static void* operator new[](size_t size)
    {
        std::cout << "Масив: " << size << " байт\n";
        return std::malloc(size);
    }
    
    static void operator delete[](void* ptr)
    {
        std::cout << "Звільнено масив\n";
        std::free(ptr);
    }
};

int main()
{
    Tracked* arr = new Tracked[10];
    delete[] arr;
    return 0;
}
```

## З додатковими параметрами (placement new)

```c++
class MemoryPool
{
    char buffer[1024];
    size_t used = 0;
    
public:
    void* alloc(size_t size)
    {
        if (used + size > 1024) throw std::bad_alloc();
        void* ptr = buffer + used;
        used += size;
        return ptr;
    }
};

class Object
{
    int x;
    
public:
    Object(int v) : x{v} {}
    
    void* operator new(size_t size, MemoryPool& pool)
    {
        return pool.alloc(size);
    }
    
    static void operator delete(void* ptr, MemoryPool& pool)
    {
        // pool не звільняє окремі об'єкти
    }
};

int main()
{
    MemoryPool pool;
    Object* obj = new(pool) Object(42);  // placement new
    
    // obj->~Object();  // вручну викликати деструктор
    // pool не потребує звільнення
    
    return 0;
}
```

## Відстеження витоків

```c++
struct AllocationInfo
{
    void* ptr;
    size_t size;
    const char* file;
    int line;
};

static std::vector<AllocationInfo> allocations;

class LeakTracker
{
public:
    static void* operator new(size_t size, const char* file, int line)
    {
        void* ptr = std::malloc(size);
        allocations.push_back({ptr, size, file, line});
        return ptr;
    }
    
    static void operator delete(void* ptr)
    {
        auto it = std::find_if(allocations.begin(), allocations.end(),
            [ptr](const auto& a) { return a.ptr == ptr; });
        if (it != allocations.end())
            allocations.erase(it);
        std::free(ptr);
    }
    
    static void report()
    {
        for (const auto& a : allocations)
            std::cout << a.file << ":" << a.line << " — " << a.size << " байт\n";
    }
};

// Макрос для зручності:
#define NEW new(__FILE__, __LINE__)

int main()
{
    LeakTracker* obj = NEW LeakTracker();
    // delete obj;  // забули!
    
    LeakTracker::report();  // покаже витік
    return 0;
}
```

## Правило

Перевантажуйте new/delete тільки якщо знаєте, що робите. В стандартному коді зазвичай достатньо звичайних версій.
