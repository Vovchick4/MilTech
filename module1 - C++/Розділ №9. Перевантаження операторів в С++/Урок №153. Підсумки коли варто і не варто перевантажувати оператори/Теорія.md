# Урок №153. Підсумки: коли варто і не варто перевантажувати оператори

## Коли ВАРТО перевантажувати

### 1. Природна семантика

```c++
// Добре: математичні операції
Money operator+(Money a, Money b);   // гроші + гроші = гроші
Vector operator*(Vector v, double s); // вектор * скаляр

// Добре: порівняння
bool operator<(Person a, Person b);  // сортування людей

// Добре: виведення
std::ostream& operator<<(std::ostream&, const Point&);
```

### 2. Аналогія з вбудованими типами

```c++
class MyString
{
public:
    MyString& operator+=(const MyString&);  // як std::string
    char operator[](int);                     // як масив
};
```

### 3. STL сумісність

```c++
class MyContainer
{
    Iterator begin();
    Iterator end();
    int& operator[](int);
};
```

## Коли НЕ ВАРТО

### 1. Неочевидна поведінка

```c++
// Погано: що означає Database - Database?
Database operator-(Database a, Database b);

// Погано: Employee * Employee?
Employee operator*(Employee a, Employee b);
```

### 2. Зміна семантики

```c++
// Погано: + віднімає
Money operator+(Money a, Money b) { return Money(a.getValue() - b.getValue()); }

// Погано: == порівнює за іншим критерієм
bool operator==(Person a, Person b) { return a.getAge() == b.getAge(); }  // ігнорує ім'я!
```

### 3. Порушення очікувань

```c++
// Погано: << не виводить
std::ostream& operator<<(std::ostream& os, const Data& d)
{
    d.saveToFile("data.txt");  // побічний ефект!
    return os;
}
```

## Керівництво

| Оператор | Варто | Не варто |
|----------|-------|----------|
| + - * / | Математичні типи | Бізнес-логіка |
| ==, < | Порівняння об'єктів | Коли критерій неоднозначний |
| <<, >> | Виведення/читання | Операції з побічними ефектами |
| [] | Контейнери | Одиночні значення |
| () | Функтори | Звичайні класи |
| -> | Розумні вказівники | Звичайні класи |

## Приклад: поганий дизайн

```c++
class Logger
{
public:
    void operator<<(const std::string& msg)  // Погано!
    {
        log(msg);
    }
};

Logger log;
log << "Помилка";  // заплутано
```

## Приклад: хороший дизайн

```c++
class Logger
{
public:
    Logger& log(const std::string& msg)  // Добре: звичайний метод
    {
        std::cout << msg;
        return *this;
    }
};

// Або:
class LogStream
{
    std::ostream* stream;
public:
    LogStream(std::ostream& os) : stream{&os} {}
    
    template<typename T>
    LogStream& operator<<(const T& value)  // Добре: природно
    {
        *stream << value;
        return *this;
    }
};
```

## Принцип найменшого здивування

**Ключове правило**: перевантажений оператор має робити те, що очікує користувач.

```c++
class Matrix
{
public:
    Matrix operator+(const Matrix& other);  // Очікувано: додавання матриць
    Matrix operator*(const Matrix& other);  // Очікувано: множення матриць
    Vector operator*(const Vector& v);      // Очікувано: добуток на вектор
};
```
