# Урок №138. Вступ до перевантаження операторів

Перевантаження операторів дозволяє визначити поведінку стандартних операторів (+, -, *, ==, тощо) для власних типів.

## Що можна перевантажувати

| Можна | Не можна |
|-------|----------|
| + - * / % | :: (роздільник області) |
| == != < > <= >= | . (крапка) |
| [] () -> | .* (вказівник на член) |
| ++ -- | ?: (тернарний) |
| << >> | sizeof |
| = & | typeid |
| += -= |= |= || | |
| && || | |
| ~ & | ^ | |
| new delete | |

## Синтаксис

```c++
class Vector2
{
    double x, y;
    
public:
    Vector2(double x, double y) : x{x}, y{y} {}
    
    // Оператор як метод класу
    Vector2 operator+(const Vector2& other) const
    {
        return Vector2(x + other.x, y + other.y);
    }
    
    // Дружня функція
    friend Vector2 operator-(const Vector2& a, const Vector2& b);
};

Vector2 operator-(const Vector2& a, const Vector2& b)
{
    return Vector2(a.x - b.x, a.y - b.y);
}
```

## Чому перевантажувати?

```c++
Vector2 a(1, 2), b(3, 4);

// Без перевантаження:
Vector2 c = add(a, b);  // функціональний стиль

// З перевантаженням:
Vector2 c = a + b;      // інтуїтивно зрозуміло
```

## Обмеження

1. Не можна змінювати пріоритет оператора
2. Не можна змінювати арність (кількість операндів)
3. Не можна створювати нові оператори
4. Не можна перевантажувати оператори для вбудованих типів

## Рекомендації

- Перевантажуйте тільки якщо це природно
- Зберігайте очікувану семантику
- Дотримуйтесь очікуваної поведінки

```c++
// Добре: money + money = money
Money operator+(const Money& a, const Money& b);

// Погано: "+" віднімає — заплутано!
Money operator+(const Money& a, const Money& b) { return Money(a.value - b.value); }
```
