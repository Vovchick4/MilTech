# Урок №143. Правило трьох (Rule of Three)

**Правило трьох**: якщо класу потрібен власний деструктор, конструктор копіювання або оператор присвоювання — то, швидше за все, потрібні всі три.

## Чому так?

Якщо клас керує ресурсами (динамічна пам'ять, файли), стандартні версії роблять поверхневе копіювання, що призводить до помилок.

## Проблема

```c++
class Buffer
{
    int* data;
    int size;
    
public:
    Buffer(int s) : size{s}
    {
        data = new int[size];
    }
    
    // ~Buffer() — автоматичний: видалить data
    
    // Buffer(const Buffer&) — автоматичний: скопіює data (поверхнево)
    // operator= — автоматичний: скопіює data (поверхнево)
};
```

## Правильне рішення

```c++
class Buffer
{
    int* data;
    int size;
    
public:
    // Конструктор
    Buffer(int s) : size{s}
    {
        data = new int[size]{};  // заповнення нулями
    }
    
    // Деструктор
    ~Buffer()
    {
        delete[] data;
    }
    
    // Конструктор копіювання
    Buffer(const Buffer& other) : size{other.size}
    {
        data = new int[size];
        for (int i = 0; i < size; ++i)
            data[i] = other.data[i];
    }
    
    // Оператор присвоювання
    Buffer& operator=(const Buffer& other)
    {
        if (this == &other)
            return *this;
        
        delete[] data;
        
        size = other.size;
        data = new int[size];
        for (int i = 0; i < size; ++i)
            data[i] = other.data[i];
        
        return *this;
    }
};
```

## Ідіома copy-and-swap

```c++
class Buffer
{
    int* data;
    int size;
    
    void swap(Buffer& other) noexcept
    {
        std::swap(data, other.data);
        std::swap(size, other.size);
    }
    
public:
    Buffer(int s) : data{new int[s]{}}, size{s} {}
    
    ~Buffer() { delete[] data; }
    
    Buffer(const Buffer& other)
        : data{new int[other.size]{}}
        , size{other.size}
    {
        for (int i = 0; i < size; ++i)
            data[i] = other.data[i];
    }
    
    Buffer& operator=(Buffer other)  // передача за значенням = копія
    {
        swap(other);
        return *this;
    }
};
```

## Приклад: файл

```c++
class FileHandler
{
    FILE* file;
    std::string name;
    
public:
    FileHandler(const std::string& filename, const char* mode)
        : name{filename}
    {
        file = fopen(filename.c_str(), mode);
    }
    
    ~FileHandler()
    {
        if (file) fclose(file);
    }
    
    FileHandler(const FileHandler& other) = delete;  // заборона копіювання
    FileHandler& operator=(const FileHandler& other) = delete;
    
    // Або власна реалізація:
    // FileHandler(const FileHandler& other) : name{other.name}
    // {
    //     file = fopen(name.c_str(), "r");  // відкриваємо заново
    // }
};
```

## Коли правило не потрібне?

```c++
// Жоден з цих класів НЕ потребує правила трьох:
class Person
{
    std::string name;  // string сам керує пам'яттю
    int age;
};

class Config
{
    std::vector<int> data;  // vector сам керує
    std::string name;
};
```

## Правило

Якщо пишете деструктор вручну — додайте конструктор копіювання і оператор присвоювання (або заборонить їх через `= delete`).
