# Урок №40. const, constexpr і символьні константи

## Константи (const)

`const` (constant) — це ключове слово, яке робить змінну незмінною після ініціалізації:

```c++
const double PI = 3.14159;
const int DAYS_IN_WEEK = 7;

PI = 3.14;  // ПОМИЛКА! const не можна змінювати
```

### Переваги const

1. **Захист від помилок** — випадкова зміна заблокована компілятором
2. **Само документування** — константа має ім'я
3. **Оптимізація** — компілятор може оптимізувати const-значення

### const і вказівники

```c++
int value = 5;
const int* ptr1 = &value;    // вказівник на const int (не можна змінити *ptr1)
int* const ptr2 = &value;    // const вказівник (не можна змінити ptr2)
const int* const ptr3 = &value; // const вказівник на const int
```

### const параметри функцій

```c++
void printValue(const int x)  // x не можна змінити всередині
{
    // x = 10;  // ПОМИЛКА!
    std::cout << x;
}
```

## constexpr (C++11)

`constexpr` — обчислюється на етапі компіляції, а не виконання:

```c++
constexpr int square(int x)
{
    return x * x;
}

int arr[square(5)];  // OK! square(5) = 25 — відомо на етапі компіляції
// int arr[func()];  // ПОМИЛКА! якщо func() не constexpr
```

### const vs constexpr

```c++
const int x = 5;          // може бути обчислено під час виконання
constexpr int y = 5;      // гарантовано на етапі компіляції

const int getValue() { return 5; }       // runtime
constexpr int getValue2() { return 5; }  // compile-time
```

### constexpr функції

```c++
constexpr int factorial(int n)
{
    return (n <= 1) ? 1 : n * factorial(n - 1);
}

int main()
{
    constexpr int fac10 = factorial(10);  // обчислюється під час компіляції
    return 0;
}
```

## Символьні константи (#define — застарілий спосіб)

```c++
#define PI 3.14159          // C-style, уникайте!
const double PI = 3.14159;  // C++ style
constexpr double PI = 3.14159;  // C++11 style (найкраще)
```

## Правила використання

1. Завжди використовуйте `const` для значень, які не повинні змінюватися
2. Використовуйте `constexpr` для констант, відомих на етапі компіляції
3. Ніколи не використовуйте `#define` для констант
4. Називайте константи в `UPPER_CASE` або з префіксом `k`:

```c++
const int MAX_SIZE = 100;   // UPPER_CASE
constexpr int kBufferSize = 256;  // k-префікс
```

## Приклад

```c++
#include <iostream>

constexpr int getArraySize() { return 100; }

int main()
{
    const int students = 30;              // runtime const
    constexpr int maxScore = 100;         // compile-time const
    
    int scores[getArraySize()];           // constexpr дозволяє це
    
    std::cout << "Розмір масиву: " << sizeof(scores) / sizeof(scores[0]);
    return 0;
}
```
