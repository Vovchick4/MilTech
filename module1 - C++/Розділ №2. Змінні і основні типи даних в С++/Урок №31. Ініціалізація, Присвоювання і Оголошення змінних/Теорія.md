# Урок №31. Ініціалізація, Присвоювання і Оголошення змінних

## Оголошення змінних

`Оголошення змінної` — це створення змінної з вказанням її типу та імені. Оголошення повідомляє компілятору, що змінна існує і який тип даних вона зберігає:

```c++
int x;          // оголошення цілочисельної змінної x
double y;       // оголошення змінної з плаваючою крапкою y
char letter;    // оголошення символьної змінної letter
bool flag;      // оголошення логічної змінної flag
```

## Ініціалізація

`Ініціалізація` — це надання змінній початкового значення в момент її створення. У C++ існує кілька форм ініціалізації:

### Копіювальна ініціалізація (copy initialization)

```c++
int x = 5;          // копіювальна ініціалізація
double pi = 3.14;
char c = 'A';
```

### Пряма ініціалізація (direct initialization)

```c++
int x(5);           // пряма ініціалізація
double pi(3.14);
```

### Uniform-ініціалізація (brace initialization) — рекомендована

```c++
int x { 5 };        // direct brace initialization
double pi { 3.14 };
int y {};           // value-initialization (x = 0)
int z { 0 };        // zero-initialization
```

Uniform-ініціалізація вважається найбезпечнішою, оскільки не дозволяє звужувальних конвертацій (narrowing conversions):

```c++
int x { 5.5 };      // ПОМИЛКА КОМПІЛЯЦІЇ! double не можна неявно звузити до int
int y = 5.5;        // OK, але y = 5 (втрата даних!)
```

### Value-ініціалізація

Порожні фігурні дужки `{}` ініціалізують змінну нульовим значенням:

```c++
int x {};           // x = 0
double d {};        // d = 0.0
char c {};          // c = '\0'
bool b {};          // b = false
```

## Присвоювання

`Присвоювання` — це надання нового значення вже існуючій змінній:

```c++
int x { 5 };        // ініціалізація (створення)
x = 10;             // присвоювання (зміна значення)
x = x + 3;          // присвоювання: x стає 13
```

### Присвоювання з операцією (compound assignment)

```c++
int x { 5 };
x += 3;             // x = x + 3 → x = 8
x -= 2;             // x = x - 2 → x = 6
x *= 4;             // x = x * 4 → x = 24
x /= 3;             // x = x / 3 → x = 8
x %= 5;             // x = x % 5 → x = 3
```

## Неініціалізовані змінні

Використання неініціалізованої змінної призводить до **невизначеної поведінки** (undefined behavior):

```c++
int x;
std::cout << x;     // НЕБЕЗПЕЧНО! x містить "сміття" (будь-яке значення)
```

Невизначена поведінка може проявлятися по-різному:
- Програма може вивести випадкове значення
- Програма може працювати "нормально" (найнебезпечніший випадок, оскільки помилка неочевидна)
- Програма може аварійно завершитися

## Правила

1. **Завжди ініціалізуйте змінні при оголошенні** — це запобігає невизначеній поведінці
2. **Використовуйте uniform-ініціалізацію `{}`** — вона безпечніша
3. **Оголошуйте змінні якомога ближче до місця їх першого використання**

## Приклад програми

```c++
#include <iostream>

int main()
{
    int x { 5 };             // ініціалізація
    std::cout << x << "\n";  // 5
    
    x = 7;                   // присвоювання
    std::cout << x << "\n";  // 7
    
    int y {};                // value-ініціалізація (0)
    std::cout << y << "\n";  // 0
    
    return 0;
}
```
