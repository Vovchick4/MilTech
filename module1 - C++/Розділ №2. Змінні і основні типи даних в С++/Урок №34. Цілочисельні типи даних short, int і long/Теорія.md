# Урок №34. Цілочисельні типи даних: short, int і long

Цілочисельні типи (integer types) зберігають цілі числа без дробової частини.

## Основні цілочисельні типи

| Тип | Типовий розмір | Діапазон (signed) |
|-----|---------------|-------------------|
| `short` | 2 байти | -32 768 до 32 767 |
| `int` | 4 байти | -2 147 483 648 до 2 147 483 647 |
| `long` | 4 або 8 байтів | залежить від платформи |
| `long long` | 8 байтів | -9 223 372 036 854 775 808 до 9 223 372 036 854 775 807 |

## signed vs unsigned

За замовчуванням цілочисельні типи зі знаком (signed):

```c++
signed int x = -5;   // може бути від'ємним
int y = -5;          // те саме, signed за замовчуванням
unsigned int z = 5;  // тільки невід'ємні (0 та додатні)
unsigned u = 10;     // скорочення: unsigned int
```

## Коли використовувати який тип?

```c++
// short — коли потрібно економити пам'ять і значення невеликі
short temperature;     // -32K до 32K

// int — стандартний вибір для цілих чисел
int age;               // більшість випадків
int count;             // лічильники, індекси

// long — коли int недостатньо
long population;        // населення країни

// long long — для дуже великих чисел
long long worldPopulation;  // ≈ 8 мільярдів
```

## Переповнення (overflow)

Якщо значення виходить за межі діапазону, відбувається переповнення:

```c++
#include <iostream>
#include <limits>

int main()
{
    int max = std::numeric_limits<int>::max();  // 2 147 483 647
    std::cout << "Max int: " << max << "\n";
    std::cout << "Max + 1: " << max + 1 << "\n";  // переповнення!
    // Виведе: -2 147 483 648 (від'ємне значення — wrapping)
    return 0;
}
```

Для unsigned типів переповнення працює як модульна арифметика:

```c++
unsigned int x = 0;
x = x - 1;  // x стає 4 294 967 295 (wrapping)
```

## Літерали цілих чисел

```c++
int d = 42;           // десятковий
int o = 052;          // вісімковий (починається з 0)
int h = 0x2A;         // шістнадцятковий (починається з 0x)
int b = 0b101010;     // двійковий (C++14+)

// Суфікси для вказання типу
unsigned int u = 42U;
long l = 42L;
long long ll = 42LL;
unsigned long long ull = 42ULL;
```

## Правила вибору

1. За замовчуванням використовуйте `int`
2. Використовуйте `unsigned` тільки для бітових масок або коли від'ємні значення неможливі
3. Уникайте `short` (крім масивів з мільйонами елементів)
4. Використовуйте `long long` для чисел > 2 мільярдів
5. Ніколи не використовуйте `long` (він різний на різних платформах)
