# Урок №35. Фіксований розмір цілочисельних типів даних

## Проблема: розмір залежить від платформи

Як ми бачили на попередньому уроці, розміри `int`, `long` тощо залежать від платформи. Наприклад, `int` може бути 2 або 4 байти. Це створює проблеми з переносимістю коду.

## Рішення: фіксовані цілочисельні типи (з inttypes.h / cstdint)

C++11 ввів типи з фіксованим розміром у заголовку `<cstdint>`:

```c++
#include <cstdint>

int8_t      // точно 8 бітів (1 байт) зі знаком
uint8_t     // точно 8 бітів без знаку
int16_t     // точно 16 бітів (2 байти) зі знаком
uint16_t    // точно 16 бітів без знаку
int32_t     // точно 32 біти (4 байти) зі знаком
uint32_t    // точно 32 біти без знаку
int64_t     // точно 64 біти (8 байтів) зі знаком
uint64_t    // точно 64 біти без знаку
```

## Приклад використання

```c++
#include <iostream>
#include <cstdint>

int main()
{
    int32_t x { 42 };        // точно 4 байти
    uint64_t y { 12345 };    // точно 8 байтів, тільки додатні
    
    std::cout << "x = " << x << " (" << sizeof(x) << " bytes)\n";
    std::cout << "y = " << y << " (" << sizeof(y) << " bytes)\n";
    
    std::cout << "Мінімум int32_t: " << INT32_MIN << "\n";
    std::cout << "Максимум int32_t: " << INT32_MAX << "\n";
    return 0;
}
```

## Найшвидші та найменші типи

```c++
#include <cstdint>

int_fast8_t     // найшвидший цілочисельний тип ≥ 8 бітів
int_fast16_t    // найшвидший тип ≥ 16 бітів
int_least8_t    // найменший (по пам'яті) тип ≥ 8 бітів
int_least16_t   // найменший тип ≥ 16 бітів
```

## Проблема з int8_t і uint8_t

`int8_t` і `uint8_t` часто є псевдонімами для `signed char` / `unsigned char`:

```c++
int8_t x { 65 };
std::cout << x;  // може вивести 'A' (символ), а не 65!
```

## Коли використовувати?

| Тип | Коли використовувати |
|-----|---------------------|
| `int` | Загальне використання, лічильники |
| `std::int32_t` | Точний розмір має значення |
| `std::int64_t` | Великі числа |
| `std::size_t` | Розміри масивів, індексація |

## Константи у <cstdint>

```c++
#include <cstdint>

int main()
{
    std::cout << INT8_MIN << " to " << INT8_MAX << "\n";
    std::cout << UINT8_MAX << "\n";
    std::cout << INT16_MIN << " to " << INT16_MAX << "\n";
    std::cout << INT32_MIN << " to " << INT32_MAX << "\n";
    std::cout << INT64_MIN << " to " << INT64_MAX << "\n";
    return 0;
}
```

## Правило

Використовуйте типи з фіксованим розміром (`int32_t`, `int64_t`) коли:
- Працюєте з бінарними форматами даних
- Потрібна точна переносимість між платформами
- Пишете мережеві протоколи або файлові формати

Для загального використання достатньо `int`.
