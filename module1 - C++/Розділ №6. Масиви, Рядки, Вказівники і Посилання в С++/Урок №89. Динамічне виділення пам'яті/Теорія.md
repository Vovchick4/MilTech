# Урок №89. Динамічне виділення пам'яті

`Динамічне виділення пам'яті` — виділення пам'яті під час виконання програми.

## Стек vs Купа (Heap)

```c++
// Стек: автоматичне керування
int x = 5;  // автоматично звільняється при виході з блоку

// Купа: ручне керування
int* ptr = new int(5);  // виділення
delete ptr;              // звільнення (обов'язково!)
```

## new та delete

```c++
int* p = new int;       // виділення пам'яті для int (неініціалізований)
int* p2 = new int(5);   // виділення + ініціалізація
int* p3 = new int{5};   // uniform-ініціалізація

delete p;               // звільнення пам'яті
delete p2;
delete p3;
```

## Динамічний масив

```c++
int size;
std::cin >> size;

int* arr = new int[size];  // масив з size елементів

for (int i = 0; i < size; ++i)
    arr[i] = i * 10;

delete[] arr;  // звільнення масиву (обов'язково з []!)
```

## Використання

```c++
int main()
{
    int n = 0;
    std::cout << "Скільки чисел? ";
    std::cin >> n;
    
    int* arr = new int[n];
    
    for (int i = 0; i < n; ++i)
    {
        std::cout << "Число " << i + 1 << ": ";
        std::cin >> arr[i];
    }
    
    int sum = 0;
    for (int i = 0; i < n; ++i) sum += arr[i];
    std::cout << "Сума: " << sum << "\n";
    
    delete[] arr;
    return 0;
}
```

## Виточка пам'яті (memory leak)

```c++
void leak()
{
    int* p = new int(5);
    // забули delete! — пам'ять не звільнена
}
// Пам'ять "витекла" — недоступна, але не звільнена
```

## Правила

1. Кожному `new` відповідає `delete`
2. Кожному `new[]` відповідає `delete[]`
3. Після `delete` скидайте вказівник на `nullptr`
4. Перевіряйте, чи не `nullptr` перед розіменуванням

## Сучасна альтернатива

```c++
// Замість new/delete — використовуйте:
std::vector<int> vec(size);  // динамічний масив (автоматичне керування)
std::unique_ptr<int> ptr = std::make_unique<int>(5);  // один об'єкт
```

## Правило

Уникайте ручного `new`/`delete`. Використовуйте `std::vector`, `std::unique_ptr`, `std::shared_ptr`.
