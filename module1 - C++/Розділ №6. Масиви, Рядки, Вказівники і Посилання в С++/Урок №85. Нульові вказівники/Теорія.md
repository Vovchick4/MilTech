# Урок №85. Нульові вказівники

Нульовий вказівник (null pointer) — вказівник, який не вказує на жоден об'єкт.

## nullptr (C++11)

```c++
int* ptr = nullptr;  // нульовий вказівник
```

До C++11 використовували `NULL` або `0`:

```c++
int* ptr1 = NULL;   // C-style, уникайте
int* ptr2 = 0;      // теж нуль
```

## Перевірка на nullptr

```c++
int* ptr = getPointer();

if (ptr != nullptr)  // або просто if (ptr)
{
    std::cout << *ptr;  // безпечно
}
else
{
    std::cout << "Вказівник нульовий";
}
```

## Розіменування nullptr — НЕБЕЗПЕЧНО!

```c++
int* ptr = nullptr;
*ptr = 5;  // ПОМИЛКА! аварійне завершення (segmentation fault)
```

## Приклад з функцією

```c++
#include <iostream>

void printValue(int* ptr)
{
    if (ptr)
        std::cout << *ptr;
    else
        std::cout << "nullptr";
}

int main()
{
    int x = 42;
    printValue(&x);        // 42
    printValue(nullptr);   // nullptr
    return 0;
}
```

## nullptr в умовах

```c++
int* ptr = getData();

if (ptr)          // те саме, що ptr != nullptr
    process(*ptr);

if (!ptr)         // те саме, що ptr == nullptr
    handleError();
```

## Небезпека "висячих" вказівників

```c++
int* ptr = new int(5);
delete ptr;
ptr = nullptr;  // Добре: після видалення скидаємо в nullptr

// Без скидання:
delete ptr;      // delete nullptr — безпечно (нічого не робить)
```

## Правило

ЗАВЖДИ ініціалізуйте вказівники. Якщо вказівник не вказує на об'єкт — присвоюйте `nullptr`. ЗАВЖДИ перевіряйте на `nullptr` перед розіменуванням.
