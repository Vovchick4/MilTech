# Урок №92. Посилання

`Посилання` (reference) — це псевдонім (альтернативне ім'я) для вже існуючої змінної.

## Оголошення

```c++
int x = 5;
int& ref = x;  // ref — посилання на x
```

ref — це не копія, а те саме, що x.

## Використання

```c++
int x = 5;
int& ref = x;

ref = 10;        // змінює x
std::cout << x;  // 10

std::cout << &x;    // адреса x
std::cout << &ref;  // та сама адреса!
```

## Посилання vs Вказівник

```c++
int x = 5;

// Вказівник
int* ptr = &x;
*ptr = 10;      // потрібен *
ptr = &other;   // можна змінити

// Посилання
int& ref = x;
ref = 10;       // як звичайна змінна
// ref = other;  // присвоює ЗНАЧЕННЯ, не переприв'язує
```

## Посилання як параметри функцій

```c++
void swap(int& a, int& b)
{
    int temp = a;
    a = b;
    b = temp;
}

int main()
{
    int x = 5, y = 10;
    swap(x, y);
    std::cout << x << " " << y;  // 10 5
    return 0;
}
```

## Посилання та великі об'єкти

```c++
struct BigData { int data[1000]; };

// Копіювання (повільно)
void processByValue(BigData d) { }

// Посилання (швидко)
void processByRef(const BigData& d) { }
```

## Повернення посилання

```c++
int& getElement(int* arr, int index)
{
    return arr[index];
}

int main()
{
    int arr[5] = {1, 2, 3, 4, 5};
    getElement(arr, 2) = 10;  // arr[2] = 10
    return 0;
}
```

## Посилання на const

```c++
int x = 5;
const int& ref = x;  // const посилання

// ref = 10;  // ПОМИЛКА! не можна змінити через const ref
x = 10;       // OK: зміна через оригінал

// const посилання можна прив'язати до тимчасового об'єкта:
const int& r = 42;  // OK
// int& r2 = 42;    // ПОМИЛКА!
```

## Обмеження посилань

```c++
int& ref;        // ПОМИЛКА! має бути ініціалізоване
int& ref = 5;    // ПОМИЛКА! не можна до тимчасового (крім const)
int& ref = &x;   // ПОМИЛКА! адреса — це вказівник
```

## Правило

Використовуйте посилання замість вказівників, коли немає потреби в `nullptr` або динамічному переприв'язуванні.
