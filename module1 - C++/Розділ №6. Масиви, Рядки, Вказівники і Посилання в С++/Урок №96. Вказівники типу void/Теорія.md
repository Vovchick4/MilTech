# Урок №96. Вказівники типу void

`void*` — це вказівник на дані без вказання типу.

## Особливості

```c++
int x = 5;
double y = 3.14;
std::string s = "Hello";

void* ptr1 = &x;  // може вказувати на будь-який тип
void* ptr2 = &y;
void* ptr3 = &s;

// std::cout << *ptr1;  // ПОМИЛКА! не можна розіменувати void*
```

## Розіменування void*

Потрібно явно привести до конкретного типу:

```c++
void* ptr = &x;
int* intPtr = static_cast<int*>(ptr);
std::cout << *intPtr;  // OK
```

## Приклад

```c++
#include <iostream>

int main()
{
    int a = 42;
    double b = 3.14;
    
    void* ptr = &a;
    std::cout << *static_cast<int*>(ptr) << "\n";  // 42
    
    ptr = &b;
    std::cout << *static_cast<double*>(ptr) << "\n";  // 3.14
    
    return 0;
}
```

## Коли використовувати?

- С-сумісність (qsort, malloc)
- Низькорівневі операції з пам'яттю
- Робота з різнотипними даними

```c++
// malloc повертає void*:
int* arr = static_cast<int*>(malloc(10 * sizeof(int)));
```

## Небезпека

```c++
int x = 42;
double y = 3.14;

void* ptr = &x;
// double* dp = static_cast<double*>(ptr);  // СКЛАДНО ВІДСТЕЖИТИ ПОМИЛКУ!
// *dp = 5.5;  // запис double туди, де int — невизначена поведінка
```

## Сучасні альтернативи

```c++
// Замість void* використовуйте:
std::variant<int, double, std::string> v;  // типобезпечне об'єднання
std::any a;                                // будь-який тип
```

## Правило

Уникайте `void*` у сучасному C++. Використовуйте шаблони, `std::variant`, перевантаження функцій — вони типобезпечніші.
