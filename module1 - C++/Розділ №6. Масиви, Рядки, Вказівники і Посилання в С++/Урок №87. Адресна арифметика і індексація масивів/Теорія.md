# Урок №87. Адресна арифметика і індексація масивів

Адресна арифметика дозволяє переміщувати вказівники по пам'яті.

## Додавання до вказівника

```c++
int arr[5] = {10, 20, 30, 40, 50};
int* ptr = arr;  // адреса arr[0]

ptr = ptr + 1;   // адреса arr[1]
ptr += 2;        // адреса arr[3]
```

Важливо: `ptr + 1` зміщує адресу на `sizeof(int)` байтів, а не на 1 байт.

## Віднімання

```c++
int arr[5] = {10, 20, 30, 40, 50};
int* ptr = arr + 4;  // arr[4]

ptr = ptr - 2;       // arr[2]
```

## Різниця вказівників

```c++
int arr[5] = {10, 20, 30, 40, 50};
int* start = arr;
int* end = arr + 4;

int count = end - start;  // 4 (кількість елементів між ними)
```

## Порівняння вказівників

```c++
int arr[5] = {10, 20, 30, 40, 50};
int* ptr = arr + 3;

if (ptr > arr)         // true (ptr вказує на arr[3])
if (ptr == arr + 3)    // true
if (ptr <= arr + 5)    // true (arr + 5 — за останнім елементом)
```

## Покроковий обхід

```c++
int arr[5] = {1, 2, 3, 4, 5};

// Вперед
for (int* p = arr; p < arr + 5; ++p)
    std::cout << *p << " ";  // 1 2 3 4 5

// Назад
for (int* p = arr + 4; p >= arr; --p)
    std::cout << *p << " ";  // 5 4 3 2 1
```

## Індексація як синтаксичний цукор

```c++
arr[i]   // еквівалентно
*(arr + i)

// Навіть так працює (але не пишіть так!):
i[arr]   // еквівалентно *(i + arr) — те саме!
```

## Приклад: копіювання масиву

```c++
int src[] = {1, 2, 3, 4, 5};
int dst[5];

for (int* s = src, *d = dst; s < src + 5; ++s, ++d)
    *d = *s;
```

## Правило

Адресна арифметика автоматично враховує розмір типу. `ptr + 1` означає "наступний елемент", незалежно від типу.

## Тест

1. Як зміниться адреса при `ptr + 1`, якщо `ptr` — `int*`?
2. Як отримати кількість елементів між двома вказівниками?
3. Чому `arr[i]` еквівалентно `*(arr + i)`?
