# Урок №78. Фіксовані масиви

`Фіксований масив` — це масив, розмір якого відомий на етапі компіляції.

## Оголошення

```c++
int arr[5] = {1, 2, 3, 4, 5};  // розмір — константа
```

## Const розмір

```c++
const int SIZE = 10;
int arr[SIZE];  // OK: SIZE — константа
```

## Ініціалізація

```c++
int arr1[5] = {1, 2, 3};       // {1, 2, 3, 0, 0}
int arr2[5] = {};              // {0, 0, 0, 0, 0}
int arr3[5] = {0};             // {0, 0, 0, 0, 0}
int arr4[] = {1, 2, 3};        // arr4[3] — розмір виведено
```

## Обмеження

```c++
int n;
std::cin >> n;
int arr[n];  // НЕ ДОЗВОЛЕНО! має бути константа (VLA — не стандарт C++)
// Використовуйте std::vector для динамічного розміру
```

## Доступ до елементів

```c++
int arr[5] = {10, 20, 30, 40, 50};

// Читання
int x = arr[2];          // 30

// Запис
arr[2] = 35;

// Небезпека: вихід за межі
// arr[10] = 100;  // невизначена поведінка
```

## Масиви символів

```c++
char str[] = "Hello";     // { 'H', 'e', 'l', 'l', 'o', '\0' }
char str2[6] = "Hello";   // явний розмір
// char str3[5] = "Hello";  // ПОМИЛКА! не вистачає місця для '\0'
```

## Переваги та недоліки

| Переваги | Недоліки |
|----------|----------|
| Швидкий доступ O(1) | Фіксований розмір |
| Мінімальні накладні витрати | Не можна змінити розмір |
| Розташування в стеку | Небезпека виходу за межі |

## Правило

Використовуйте фіксовані масиви для невеликих наборів даних з відомим розміром. Для всього іншого — `std::vector` або `std::array`.

## Тест

2. Чим ініціалізується `int arr[5] = {};`?
3. Який розмір має `char str[] = "Hi";`?
