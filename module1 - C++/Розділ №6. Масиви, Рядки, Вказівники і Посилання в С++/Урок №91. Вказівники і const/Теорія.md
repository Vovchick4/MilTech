# Урок №91. Вказівники і const

`const` у поєднанні з вказівниками має кілька варіантів.

## Вказівник на const

Не можна змінити значення за адресою:

```c++
const int x = 5;
const int* ptr = &x;  // вказівник на const int

// *ptr = 10;  // ПОМИЛКА! не можна змінювати через ptr

int y = 10;
ptr = &y;  // OK: можна змінити, на що вказує
```

## const вказівник

Не можна змінити адресу (сам вказівник const):

```c++
int x = 5;
int y = 10;
int* const ptr = &x;  // const вказівник

*ptr = 10;  // OK: можна змінювати значення
// ptr = &y;  // ПОМИЛКА! не можна змінити адресу
```

## const вказівник на const

Нічого не можна змінювати:

```c++
const int x = 5;
const int* const ptr = &x;  // все const

// *ptr = 10;  // ПОМИЛКА!
// ptr = ...;  // ПОМИЛКА!
```

## Таблиця

| Оголошення | Зміна *ptr | Зміна ptr |
|------------|-----------|-----------|
| `int*` | Так | Так |
| `const int*` | Ні | Так |
| `int* const` | Так | Ні |
| `const int* const` | Ні | Ні |

## const вказівник та функції

```c++
// Вхідні дані: не змінюємо
void print(const int* arr, int size)
{
    for (int i = 0; i < size; ++i)
        std::cout << arr[i] << " ";
    // arr[0] = 5;  // ПОМИЛКА!
}

// Вихідні дані: const вказівник
int x = 5;
int* const getPtr() { return &x; }  // не можна змінити адресу
```

## const методи та вказівник

```c++
class MyClass
{
    int value;
public:
    int getValue() const   // const метод
    {
        return value;      // OK: читаємо
        // value = 5;      // ПОМИЛКА! const метод не змінює
    }
};
```

## const ітератори

```c++
const std::vector<int> vec = {1, 2, 3};
auto it = vec.begin();  // const_iterator — read-only
// *it = 5;  // ПОМИЛКА!
```

## Правило

За замовчуванням передавайте вказівники як `const`, якщо не плануєте змінювати дані. Це робить код безпечнішим і зрозумілішим.
