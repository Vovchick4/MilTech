# Урок №86. Вказівники і масиви

У C++ масиви та вказівники тісно пов'язані. Ім'я масиву — це вказівник на його перший елемент.

## Ім'я масиву — вказівник

```c++
int arr[5] = {10, 20, 30, 40, 50};

std::cout << arr;      // адреса першого елемента
std::cout << &arr[0];  // те саме
std::cout << *arr;     // 10 (перший елемент)
```

## Адресна арифметика

```c++
int arr[5] = {10, 20, 30, 40, 50};

arr[2];     // 30 — доступ через індекс
*(arr + 2); // 30 — через адресну арифметику (arr + 2 * sizeof(int))
```

## Вказівник як ітератор масиву

```c++
int arr[] = {1, 2, 3, 4, 5};
int* begin = arr;       // початок
int* end = arr + 5;     // кінець (за останнім елементом)

for (int* p = begin; p != end; ++p)
{
    std::cout << *p << " ";  // 1 2 3 4 5
}
```

## Вказівник та індекс

```c++
int arr[5] = {1, 2, 3, 4, 5};
int* ptr = arr;

ptr[0];     // 1 (еквівалентно *ptr)
ptr[2];     // 3 (еквівалентно *(ptr + 2))
*ptr = 10;  // arr[0] = 10
```

## Параметри-масиви функцій

Коли масив передається у функцію, він "розпадається" на вказівник:

```c++
void printArray(int arr[], int size)  // int arr[] == int* arr
{
    for (int i = 0; i < size; ++i)
        std::cout << arr[i] << " ";
}

// Еквівалентно:
void printArray(int* arr, int size)
{
    for (int i = 0; i < size; ++i)
        std::cout << arr[i] << " ";
}
```

## Чому sizeof(arr) в функції не працює?

```c++
void func(int arr[5])
{
    std::cout << sizeof(arr);  // 8! (розмір вказівника, не масиву)
}
```

## Вказівник vs масив

```c++
int arr[5] = {1, 2, 3, 4, 5};
int* ptr = arr;

// Масив:
// sizeof(arr) == 20 (5 * 4)
// arr — константна адреса, не можна змінити: arr++ — ПОМИЛКА

// Вказівник:
// sizeof(ptr) == 8 (розмір вказівника)
// ptr — змінна: ptr++ — OK, тепер ptr[0] == 2
```

## Правило

Ім'я масиву — це константний вказівник на перший елемент. Вказівники та індекси взаємозамінні: `arr[i]` == `*(arr + i)`.
