# Урок №127. Делегуючі конструктори

Делегуючий конструктор (C++11) викликає інший конструктор того ж класу.

## Проблема: дублювання коду

```c++
class Employee
{
    std::string name;
    int id;
    double salary;
    
public:
    Employee()  // дублювання!
    {
        name = "Невідомо";
        id = 0;
        salary = 0.0;
    }
    
    Employee(const std::string& n)  // дублювання!
    {
        name = n;
        id = 0;
        salary = 0.0;
    }
    
    Employee(const std::string& n, int i, double s)  // дублювання!
    {
        name = n;
        id = i;
        salary = s;
    }
};
```

## Рішення: делегування

```c++
class Employee
{
    std::string name;
    int id;
    double salary;
    
public:
    Employee(const std::string& n, int i, double s)
        : name{n}, id{i}, salary{s} {}  // цільовий
    
    Employee() : Employee{"Невідомо", 0, 0.0} {}  // делегує
    
    Employee(const std::string& n) : Employee{n, 0, 0.0} {}  // делегує
};
```

## Обмеження

```c++
class Example
{
public:
    Example() : Example{0}  // делегування
    {
        // код
    }
    
    Example(int x) : Example{}  // ПОМИЛКА! цикли не дозволені!
    {
        // код
    }
};
```

## Переваги

1. **Менше дублювання** — один цільовий конструктор
2. **Зручність** — різні способи створення об'єкта
3. **Підтримка** — зміни в одному місці

## Приклад

```c++
class Rectangle
{
    double w, h;
    
public:
    Rectangle(double w, double h) : w{w}, h{h} {}
    
    Rectangle() : Rectangle{1.0, 1.0} {}  // квадрат 1×1
    
    Rectangle(double side) : Rectangle{side, side} {}  // квадрат
};

// Використання:
Rectangle r1;            // 1×1
Rectangle r2(5.0);       // 5×5 (квадрат)
Rectangle r3(4.0, 3.0);  // 4×3
```

## Правило

Використовуйте делегування, щоб уникнути дублювання коду в конструкторах.
