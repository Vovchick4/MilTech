# Урок №128. Деструктори

`Деструктор` — спеціальний метод, який викликається при знищенні об'єкта.

## Синтаксис

```c++
class Example
{
public:
    ~Example()  // деструктор
    {
        // очищення
    }
};
```

## Коли викликається?

```c++
class Demo
{
public:
    Demo()  { std::cout << "Створено\n"; }
    ~Demo() { std::cout << "Знищено\n"; }
};

int main()
{
    Demo d;  // "Створено"
    // ...
}            // "Знищено" (при виході з області видимості)
```

## Типове використання: звільнення ресурсів

```c++
class IntArray
{
    int* data;
    int size;
    
public:
    IntArray(int s) : size{s}, data{new int[s]} {}
    
    ~IntArray()
    {
        delete[] data;  // звільнення пам'яті
    }
    
    int& operator[](int i) { return data[i]; }
};
```

## Деструктор для динамічних об'єктів

```c++
int main()
{
    Demo* d = new Demo();
    delete d;  // викликає деструктор
    return 0;
}
```

## Порядок знищення

Зворотний порядку створення:

```c++
class Member
{
    std::string name;
public:
    Member(const std::string& n) : name{n} {
        std::cout << "Створено " << name << "\n";
    }
    ~Member() {
        std::cout << "Знищено " << name << "\n";
    }
};

class Container
{
    Member a{"A"}, b{"B"};
};

int main()
{
    Container c;  // Створено A, Створено B
}                 // Знищено B, Знищено A
```

## Правило трьох (Rule of Three)

Якщо клас має ручне керування ресурсами, потрібні:
1. Деструктор
2. Конструктор копіювання
3. Оператор присвоювання

## Коли писати деструктор?

- Клас керує динамічною пам'яттю
- Клас керує файлами, сокетами, з'єднаннями
- Клас керує будь-яким ресурсом

## Правило

Якщо клас не потребує деструктора — не пишіть його. Компілятор створить неявний деструктор, який знищує всі поля.
