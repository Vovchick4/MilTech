# Урок №130. Класи і заголовкові файли

У багатофайлових проектах класи зазвичай розділяють на `.h` (оголошення) та `.cpp` (реалізація).

## Структура

**Point.h** — оголошення класу:

```c++
#pragma once

class Point
{
    int x, y;
    
public:
    Point(int x = 0, int y = 0);
    
    int getX() const;
    int getY() const;
    void setX(int x);
    void setY(int y);
    
    void print() const;
};
```

**Point.cpp** — реалізація:

```c++
#include "Point.h"
#include <iostream>

Point::Point(int x, int y) : x{x}, y{y} {}

int Point::getX() const { return x; }
int Point::getY() const { return y; }
void Point::setX(int x) { this->x = x; }
void Point::setY(int y) { this->y = y; }

void Point::print() const
{
    std::cout << "(" << x << ", " << y << ")";
}
```

## Чому розділяти?

1. **Швидкість компіляції** — зміна .cpp вимагає перекомпіляції лише цього файлу
2. **Приховування реалізації** — .h показує тільки інтерфейс
3. **Організація** — логічне групування

## Inline методи

Маленькі методи можна визначити прямо в заголовку:

```c++
// Point.h
class Point
{
    int x, y;
    
public:
    int getX() const { return x; }  // inline
    int getY() const { return y; }  // inline
};
```

## const та заголовки

```c++
class Config
{
    static const int MAX_SIZE = 100;        // OK: static constexpr
    static constexpr double PI = 3.14159;   // OK
};
```

## Приклад: повна програма

**Student.h:**
```c++
#pragma once
#include <string>

class Student
{
    std::string name;
    int age;
    
public:
    Student(const std::string& name, int age);
    void print() const;
};
```

**Student.cpp:**
```c++
#include "Student.h"
#include <iostream>

Student::Student(const std::string& name, int age)
    : name{name}, age{age} {}

void Student::print() const
{
    std::cout << name << " (" << age << " років)";
}
```

**main.cpp:**
```c++
#include "Student.h"

int main()
{
    Student s("Анна", 20);
    s.print();
    return 0;
}
```

## Правило

Один клас — два файли (зазвичай). Ім'я файлу = ім'я класу.
