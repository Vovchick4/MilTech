# Урок №126. Ініціалізація нестатичних членів класу

C++11 дозволяє ініціалізувати поля прямо при оголошенні (in-class initializer).

## Синтаксис

```c++
class Point
{
    int x = 0;  // in-class initializer
    int y = 0;
};
```

## Коли це корисно?

```c++
class Config
{
    std::string server = "localhost";
    int port = 8080;
    bool debug = false;
    std::vector<int> ids = {1, 2, 3};
};
```

## Пріоритет: список ініціалізації vs in-class

```c++
class Widget
{
    int width = 100;    // значення за замовчуванням
    int height = 100;
    
public:
    Widget() {}                      // width=100, height=100
    Widget(int s) : width{s}, height{s} {}  // width=s, height=s
    Widget(int w, int h) : width{w} {}      // width=w, height=100 (!)
};
```

Список ініціалізації ПЕРЕВИЗНАЧАЄ in-class значення.

## Складні типи

```c++
class Team
{
    std::string name = "Нова команда";
    std::vector<std::string> members = {"Капітан"};
    
public:
    Team() = default;
    Team(const std::string& n) : name{n} {}
};
```

## Приклад: лічильник

```c++
class Counter
{
    int count = 0;  // завжди починає з 0
    
public:
    Counter() = default;
    Counter(int start) : count{start} {}
    
    void increment() { ++count; }
    int getValue() const { return count; }
};
```

## Коли використовувати?

- Для значень за замовчуванням: `int x = 0;`
- Для константних значень у класі
- Щоб уникнути дублювання в конструкторах

## Правило

Використовуйте in-class ініціалізацію для значень за замовчуванням. Список ініціалізації використовуйте для значень, що залежать від конструктора.
