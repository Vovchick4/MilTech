# Урок №134. Дружні функції і класи

`friend` — дозволяє функції або іншому класу доступ до private членів.

## Дружня функція

```c++
class Point
{
    int x, y;
    
public:
    Point(int x, int y) : x{x}, y{y} {}
    
    friend void printPoint(const Point& p);  // дружня функція
};

// Функція має доступ до private:
void printPoint(const Point& p)
{
    std::cout << "(" << p.x << ", " << p.y << ")";
}
```

## Дружній клас

```c++
class Engine;  // попереднє оголошення

class Car
{
    int speed = 0;
    
    friend class Mechanic;  // Mechanic має доступ до private Car
    friend void boost(Car& c, int amount);  // дружня функція
};

class Mechanic
{
public:
    void repair(Car& c)
    {
        c.speed = 0;  // доступ до private!
    }
    
    void setSpeed(Car& c, int s)
    {
        c.speed = s;  // доступ до private!
    }
};

void boost(Car& c, int amount)
{
    c.speed += amount;  // доступ до private!
}
```

## Дружній метод

```c++
class Car;  // попереднє оголошення

class Mechanic
{
public:
    void repair(Car& c);  // тільки оголошення
};

class Car
{
    int speed;
    
    friend void Mechanic::repair(Car& c);  // конкретний метод
};
```

## Перевантаження операторів

Дружні функції часто використовуються для перевантаження операторів:

```c++
class Vector2
{
    double x, y;
    
public:
    Vector2(double x, double y) : x{x}, y{y} {}
    
    friend Vector2 operator+(const Vector2& a, const Vector2& b)
    {
        return Vector2(a.x + b.x, a.y + b.y);
    }
    
    friend std::ostream& operator<<(std::ostream& os, const Vector2& v)
    {
        os << "(" << v.x << ", " << v.y << ")";
        return os;
    }
};
```

## Обмеження friend

```c++
class A
{
    friend class B;  // B бачить private A
    int secret;
};

class B
{
    // A НЕ бачить private B! (friend — не взаємний)
    int bSecret;
};

// friend не наслідується:
class D : public A
{
    // D НЕ має доступу до private A
};
```

## Коли використовувати?

- Перевантаження бінарних операторів (<<, >>, +)
- Класи, які тісно пов'язані (наприклад, ітератор і контейнер)
- Операції між двома класами

## Правило

Використовуйте `friend` обережно — він порушує інкапсуляцію. Краще намагатися обійтися без нього.
