# Урок №125. Список ініціалізації членів класу

## Синтаксис

Список ініціалізації — це спосіб ініціалізувати поля класу ДО виконання тіла конструктора:

```c++
class Example
{
    int a;
    double b;
    std::string s;
    
public:
    Example(int x, double y, const std::string& str)
        : a{x}, b{y}, s{str}  // список ініціалізації
    {
        // тіло
    }
};
```

## Чому це краще?

```c++
// Погано: спочатку default-ініціалізація, потім присвоювання
class Bad
{
    std::string name;
public:
    Bad(const std::string& n) { name = n; }  // 2 операції!
};

// Добре: прямий виклик конструктора
class Good
{
    std::string name;
public:
    Good(const std::string& n) : name{n} {}  // 1 операція!
};
```

## const та посилання

const та посилання ОБОВ'ЯЗКОВО ініціалізуються в списку:

```c++
class WithConstRef
{
    const int id;
    int& ref;
    
public:
    WithConstRef(int i, int& r) : id{i}, ref{r} {}
    // id = i;  // ПОМИЛКА! const не можна присвоїти
};
```

## Порядок ініціалізації

Поля ініціалізуються в ПОРЯДКУ ЇХ ОГОЛОШЕННЯ в класі, а не в списку:

```c++
class Order
{
    int a;
    int b;
    
public:
    // НЕбезпечно: b ініціалізується ПЕРЕД a (a, b — порядок оголошення)
    Order() : b{5}, a{b} { /* a = невизначено! */ }
    // Правильно: a{5}, b{a}
};
```

## Делегування конструкторів (C++11)

```c++
class Point
{
    int x, y;
    
public:
    Point() : Point{0, 0} {}  // делегує Point(int, int)
    
    Point(int val) : Point{val, val} {}  // делегує
    
    Point(int x, int y) : x{x}, y{y} {}  // цільовий конструктор
};
```

## Приклад

```c++
class Student
{
    std::string name;
    int age;
    double grade;
    
public:
    Student() : Student{"", 0, 0.0} {}  // делегування
    
    Student(const std::string& n, int a)
        : Student{n, a, 0.0} {}  // делегування
    
    Student(const std::string& n, int a, double g)
        : name{n}, age{a}, grade{g} {}  // цільовий
};
```
