# Урок №123. Інкапсуляція, Геттери і Сеттери

`Інкапсуляція` — приховування внутрішніх даних об'єкта і надання контрольованого доступу через методи.

## Принцип

```c++
class Temperature
{
private:
    double celsius;  // приховано
    
public:
    void setCelsius(double c) { celsius = c; }
    void setFahrenheit(double f) { celsius = (f - 32) / 1.8; }
    double getCelsius() const { return celsius; }
    double getFahrenheit() const { return celsius * 1.8 + 32; }
};
```

## Чому інкапсуляція важлива?

```c++
class BadBankAccount
{
public:
    double balance;  // public — поганo!
};

BadBankAccount acc;
acc.balance = -100;  // дозволено! від'ємний баланс


class GoodBankAccount
{
private:
    double balance = 0;
    
public:
    void deposit(double amount) {
        if (amount > 0) balance += amount;
    }
    
    bool withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            return true;
        }
        return false;
    }
    
    double getBalance() const { return balance; }
};
```

## Getters (акцесори)

Методи для читання даних:

```c++
class Person
{
    std::string name;
    int age;
    
public:
    std::string getName() const { return name; }
    int getAge() const { return age; }
};
```

## Setters (мутатори)

Методи для зміни даних (з валідацією):

```c++
class Person
{
    std::string name;
    int age = 0;
    
public:
    void setName(const std::string& n)
    {
        if (!n.empty()) name = n;  // валідація
    }
    
    void setAge(int a)
    {
        if (a >= 0 && a <= 150) age = a;  // валідація
    }
};
```

## Контроль доступу

```c++
class Product
{
    std::string id;     // read-only — getter, без setter
    double price;        // read-write
    int quantity = 0;   // write-only — setter, без getter
    
public:
    std::string getId() const { return id; }  // тільки читання
    // void setId(...) — НЕМАЄ, id задається при створенні
    
    double getPrice() const { return price; }
    void setPrice(double p) { if (p > 0) price = p; }
    
    void setQuantity(int q) { quantity = q; }  // тільки запис
    // int getQuantity() — НЕМАЄ (непотрібно)
};
```

## Переваги інкапсуляції

1. **Захист даних** — контроль валідності
2. **Гнучкість** — зміна реалізації без зміни інтерфейсу
3. **Тестування** — легше тестувати через методи

## Правило
