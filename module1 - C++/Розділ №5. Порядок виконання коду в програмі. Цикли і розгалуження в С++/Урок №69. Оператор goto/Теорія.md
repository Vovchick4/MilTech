# Урок №69. Оператор goto

`goto` — це оператор безумовного переходу до мітки (label) в коді.

## Синтаксис

```c++
goto мітка;

// ... код, який пропускається

мітка:
    // код, що виконується після goto
```

## Приклад

```c++
#include <iostream>

int main()
{
    std::cout << "Початок\n";
    
    goto skip;
    
    std::cout << "Це ніколи не виконається\n";
    
skip:
    std::cout << "Після goto\n";
    
    return 0;
}
```

Виведе: "Початок" → "Після goto"

## Чому goto — це погано? ("goto вважається шкідливим")

1. **Спагеті-код** — важко відстежити потік виконання
2. **Складність налагодження** — незрозуміло, звідки прийшло виконання
3. **Порушення структурного програмування** — одна точка входу, одна точка виходу
4. **Пропуск ініціалізації** — можна "перестрибнути" через оголошення змінних

```c++
goto error;

int x = 5;  // НЕБЕЗПЕЧНО! ініціалізація пропущена

error:
    // ...
```

## Коли goto може бути виправданий?

### Вихід з вкладених циклів

```c++
// Без goto — потрібен прапорець
bool found = false;
for (int i = 0; i < 10 && !found; ++i)
{
    for (int j = 0; j < 10 && !found; ++j)
    {
        if (arr[i][j] == target)
        {
            found = true;
        }
    }
}

// З goto — простіше (хоча спірно)
for (int i = 0; i < 10; ++i)
{
    for (int j = 0; j < 10; ++j)
    {
        if (arr[i][j] == target)
            goto found;
    }
}
found:
    // ...
```

### Очищення ресурсів (C-style код)

У C++ сучасні конструкції (RAII, винятки) роблять goto непотрібним.

## Сучасні альтернативи

| Ситуація | Замість goto |
|----------|-------------|
| Вихід з вкладених циклів | `return` з функції |
| Обробка помилок | `try`/`catch` або `std::optional` |
| Очищення | RAII, деструктори |

## Приклад: заміна goto на функцію

```c++
// Замість goto — використовуйте функцію
bool processData()
{
    for (int i = 0; i < 10; ++i)
        for (int j = 0; j < 10; ++j)
            if (arr[i][j] == target)
                return true;
    return false;
}
```

## Правило

Ніколи не використовуйте `goto` у звичайному коді. Він був необхідний у ранніх мовах програмування, але в C++ є кращі альтернативи.
