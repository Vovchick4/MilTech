# Урок №76. Введення в тестування коду

Тестування — процес перевірки, що програма працює коректно.

## Чому тестування важливе?

1. **Виявлення помилок** — знайти проблеми до того, як їх знайдуть користувачі
2. **Впевненість у змінах** — після рефакторингу тести підтверджують, що все працює
3. **Документація** — тести показують, як має працювати код
4. **Дизайн** — тестований код часто краще спроектований

## Види тестування

| Тип | Опис |
|-----|------|
| **Unit-тести** | Тестування окремих функцій/класів |
| **Integration-тести** | Тестування взаємодії компонентів |
| **Manual тести** | Ручне тестування |

## Просте ручне тестування

```c++
int add(int a, int b) { return a + b; }

int main()
{
    // Прості тести
    if (add(2, 3) != 5) std::cout << "ПОМИЛКА: 2+3 != 5\n";
    if (add(-1, 1) != 0) std::cout << "ПОМИЛКА: -1+1 != 0\n";
    if (add(0, 0) != 0) std::cout << "ПОМИЛКА: 0+0 != 0\n";
    
    std::cout << "Тести завершено\n";
    return 0;
}
```

## Використання assert

```c++
#include <cassert>

int add(int a, int b) { return a + b; }

int main()
{
    assert(add(2, 3) == 5);
    assert(add(-1, 1) == 0);
    assert(add(0, 0) == 0);
    
    // Якщо всі assert пройшли — програма працює
    return 0;
}
```

## Структуроване тестування

```c++
#include <iostream>
#include <cassert>

int divide(int a, int b)
{
    assert(b != 0);  // захист від ділення на 0
    return a / b;
}

void testDivide()
{
    assert(divide(10, 2) == 5);
    assert(divide(7, 3) == 2);  // ціле ділення
    assert(divide(0, 5) == 0);
    assert(divide(-10, 2) == -5);
    // assert(divide(5, 0) == 0);  // спрацює assert!
    std::cout << "testDivide OK\n";
}

int main()
{
    testDivide();
    std::cout << "Всі тести пройдено!\n";
    return 0;
}
```

## Тестування граничних випадків

```c++
int findMax(const std::vector<int>& vec)
{
    assert(!vec.empty());
    int max = vec[0];
    for (int v : vec)
        if (v > max) max = v;
    return max;
}

void testFindMax()
{
    assert(findMax({1, 2, 3}) == 3);     // звичайний
    assert(findMax({-5, -2, -10}) == -2); // від'ємні
    assert(findMax({7}) == 7);             // один елемент
    assert(findMax({5, 5, 5}) == 5);      // однакові
    // findMax({});  // assert спрацює!
}
```

## Підхід TDD (Test-Driven Development)

1. Написати тест **до** реалізації
2. Запустити — тест падає (червоний)
3. Написати реалізацію
4. Запустити — тест проходить (зелений)
5. Рефакторинг

```c++
// 1. Тест:
void testIsEven()
{
    assert(isEven(2) == true);
    assert(isEven(3) == false);
    assert(isEven(0) == true);
    assert(isEven(-2) == true);
}

// 2. Потім реалізація:
bool isEven(int n) { return n % 2 == 0; }
```

## Тестовий фреймворк (наприклад, Catch2)

```c++
// З використанням бібліотеки Catch2:
TEST_CASE("Addition works")
{
    REQUIRE(add(2, 3) == 5);
    REQUIRE(add(-1, 1) == 0);
}

TEST_CASE("Division works")
{
    REQUIRE(divide(10, 2) == 5);
    REQUIRE(divide(7, 3) == 2);
}
```

## Правило

Тестуйте:
- Звичайні випадки
- Граничні випадки (крайові значення)
- Помилкові ситуації (некоректні вхідні дані)
