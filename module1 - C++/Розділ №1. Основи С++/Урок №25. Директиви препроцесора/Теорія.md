# Урок №25. Директиви препроцесора

`Препроцесор` — це програма, яка обробляє вихідний код перед компіляцією. Директиви препроцесора починаються з символу `#`.

### #include — вставка файлу

Найпоширеніша директива. Вставляє вміст іншого файлу в поточний:

```c++
#include <iostream>   // системні заголовки — кутові дужки
#include "myfile.h"   // власні заголовки — лапки
```

### #define — макровизначення

Створює макрос — правило заміни тексту:

```c++
#define PI 3.14159
#define MAX_STUDENTS 30

double area = PI * r * r;   // препроцесор замінить PI на 3.14159
```

Макроси можуть мати параметри (але краще використовувати функції):

```c++
#define SQUARE(x) ((x) * (x))
int y = SQUARE(5);  // заміниться на ((5) * (5))
```

### #ifdef / #ifndef / #endif — умовна компіляція

Дозволяє включати або виключати частини коду:

```c++
#define DEBUG

#ifdef DEBUG
    std::cout << "Режим налагодження" << std::endl;
#endif
```

`#ifndef` виконує код, якщо макрос НЕ визначений:

```c++
#ifndef NDEBUG
    // код для налагодження
#endif
```

### #pragma once — альтернатива header guards

Вказує компілятору включити файл лише один раз:

```c++
#pragma once
// решта заголовкового файлу
```

### Сучасна практика

- Для констант використовуйте `const`/`constexpr` замість `#define`
- Для заголовкових файлів використовуйте `#pragma once` або header guards
- `#include` використовуйте для підключення заголовків

### Правило: Уникайте #define для констант і макросів-функцій — використовуйте сучасні можливості C++.
