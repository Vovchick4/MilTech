# Урок №22. Прототип функції і Попереднє оголошення

### Проблема порядку визначення

Компілятор читає код зверху вниз. Якщо викликати функцію до її визначення — буде помилка:

```c++
int main()
{
    foo();  // ПОМИЛКА! foo() ще не визначена
    return 0;
}

void foo()  // визначення після виклику
{
    std::cout << "Hello";
}
```

### Прототип функції

`Прототип функції` (попереднє оголошення) — це оголошення функції без її тіла, яке повідомляє компілятору: "ця функція існує, її визначення буде пізніше".

```c++
// Прототип (попереднє оголошення)
void foo();

int main()
{
    foo();  // OK! компілятор знає про foo
    return 0;
}

// Визначення функції
void foo()
{
    std::cout << "Hello";
}
```

### Синтаксис прототипу

Прототип копіює заголовок функції і закінчується крапкою з комою:

```c++
// Прототипи
int add(int a, int b);
double multiply(double x, double y);
void printMessage(std::string msg);
bool isValid(int age);

// Визначення можуть бути де завгодно (навіть в іншому файлі)
int add(int a, int b)
{
    return a + b;
}
```

Імена параметрів у прототипі можна опускати:

```c++
int add(int, int);  // теж коректний прототип
```

### Для чого це потрібно?

1. **Організація коду** — прототипи часто розміщують на початку файлу або в заголовкових файлах
2. **Багатофайлові проекти** — прототипи в `.h` файлах дозволяють використовувати функції з інших `.cpp` файлів
3. **Взаємозалежні функції** — коли функція A викликає B, а B викликає A

### Правило: Завжди додавайте прототипи функцій, якщо визначення знаходиться після точки виклику.
