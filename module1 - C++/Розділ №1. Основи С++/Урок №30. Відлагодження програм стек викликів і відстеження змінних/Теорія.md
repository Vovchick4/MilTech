# Урок №30. Відлагодження програм: стек викликів і відстеження змінних

### Стек викликів (Call Stack)

`Стек викликів` — це структура даних, яка відстежує порядок виклику функцій у програмі. Кожного разу, коли викликається функція, в стек додається новий `фрейм` (запис). Коли функція завершується, її фрейм видаляється.

```c++
void funcC() { /* ... */ }
void funcB() { funcC(); }    // B викликає C
void funcA() { funcB(); }    // A викликає B

int main()
{
    funcA();  // стек: main → funcA → funcB → funcC
    return 0;
}
```

Стек викликів виглядає так:
```
funcC()        ← поточна функція
funcB()
funcA()
main()
```

### Перегляд стеку викликів у дебагері

Під час зупинки на точці зупину можна подивитися стек викликів. Це показує, яка функція викликала яку, і де саме виконання було зупинено.

Це допомагає відповісти на питання: "Як ми опинилися в цьому місці коду?"

### Відстеження змінних

Дебагер дозволяє спостерігати за значеннями змінних у реальному часі:

- **Watch (спостереження)** — додайте змінну, і дебагер показуватиме її поточне значення
- **Hover (наведення)** — наведіть курсор на змінну в коді, щоб побачити її значення
- **Autos/Locals** — автоматичне відображення локальних змінних

```c++
int main()
{
    int x = 5;           // Watch: x = 5
    x = x + 3;           // Watch: x = 8
    int y = x * 2;       // Watch: x = 8, y = 16

    for (int i = 0; i < 3; i++)  // Watch: i = 0, 1, 2
    {
        y = y + i;
    }
    return 0;
}
```

### Використання стеку викликів і змінних разом

Найпотужніша техніка: поставити точку зупину, подивитися стек викликів (щоб зрозуміти контекст) і перевірити значення змінних (щоб знайти логічну помилку).

### Умовні точки зупину

Можна вказати умову — програма зупиниться лише коли умова істинна:

```
Наприклад: breakpoint на рядку з x = i, умова: i == 10
```

Це корисно в циклах — не потрібно крокувати 100 разів.

### Правило: Стек викликів показує "де ви знаходитесь", а значення змінних — "що відбувається".
