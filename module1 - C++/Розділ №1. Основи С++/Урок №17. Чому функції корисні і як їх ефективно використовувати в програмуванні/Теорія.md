# Урок №17. Чому функції корисні і як їх ефективно використовувати в програмуванні?

### Чому функції корисні?

1. **Уникнення дублювання коду (DRY)** — один раз написали, багато разів використали:

```c++
// Без функції — дублювання
std::cout << "Площа: " << 5 * 3 << "\n";
std::cout << "Площа: " << 7 * 2 << "\n";
std::cout << "Площа: " << 4 * 6 << "\n";

// З функцією
int area(int w, int h) { return w * h; }
std::cout << "Площа: " << area(5, 3) << "\n";
std::cout << "Площа: " << area(7, 2) << "\n";
std::cout << "Площа: " << area(4, 6) << "\n";
```

2. **Модульність** — код розбивається на логічні блоки, кожен з яких відповідає за одне завдання.

3. **Читабельність** — назва функції пояснює, що робить код:

```c++
if (isGameOver(score))  // зрозуміло без коментарів
```

4. **Тестування** — функції легко тестувати окремо.

5. **Повторне використання** — функцію можна використовувати в різних програмах.

### Як ефективно використовувати функції?

#### Одне завдання — одна функція

Функція повинна робити тільки одне завдання:

```c++
// Погано: функція робить забагато
void processStudent() {
    // ввід даних
    // обчислення оцінки
    // виведення результату
    // збереження у файл
}

// Добре: розбиваємо на окремі функції
void inputData();
double calculateGrade();
void printResult();
void saveToFile();
```

#### Довжина функції

Функція має бути досить короткою, щоб поміщатися на екрані (20-30 рядків). Якщо більше — розбивайте.

#### Імена функцій

Ім'я має бути дієсловом або фразою, яка описує дію:

```c++
getAverage()     // OK
calculateTotal() // OK
isValid()        // OK
func1()          // погано
doStuff()        // погано
```

### Правило: Функція повинна виконувати одне завдання і робити це добре.
