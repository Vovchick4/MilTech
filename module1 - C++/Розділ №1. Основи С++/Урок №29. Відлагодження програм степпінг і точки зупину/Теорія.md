# Урок №29. Відлагодження програм: степпінг і точки зупину

`Відлагодження` (debugging) — це процес пошуку та виправлення помилок у програмі. Існує два підходи: "грубий" (друк повідомлень) і "розумний" (використання дебагера).

### Друк повідомлень (грубий підхід)

Найпростіший спосіб — додати `std::cout` для відстеження:

```c++
int add(int a, int b)
{
    std::cout << "add: a=" << a << ", b=" << b << std::endl;
    return a + b;
}
```

Недоліки: потрібно видаляти після налагодження, засмічує код.

### Дебагер — розумний підхід

`Дебагер` — це інструмент, який дозволяє контролювати виконання програми. Основні можливості:

- **Точки зупину** (breakpoints) — зупинка програми в певному рядку
- **Степпінг** (stepping) — покрокове виконання
- **Перегляд змінних** — спостереження за значеннями

### Точки зупину (Breakpoints)

`Точка зупину` позначає рядок, де програма призупинить виконання. Встановлюється кліком біля номера рядка в IDE.

```c++
int main()
{
    int x = 5;      // ← точка зупину: програма зупиниться тут
    x = x + 3;      // можна побачити зміну x
    std::cout << x; // і переконатися, що виводиться правильне значення
    return 0;
}
```

### Степпінг (покрокове виконання)

Після зупинки на точці зупину можна крокувати:

- **Step Into (F11)** — увійти у виклик функції
- **Step Over (F10)** — виконати рядок, не заходячи у функції
- **Step Out (Shift+F11)** — вийти з поточної функції

```c++
int main()
{
    int x = 5;          // ← точка зупину
    int y = add(x, 3);   // Step Into → перейде в add()
                         // Step Over → виконає add() і перейде далі
    std::cout << y;
    return 0;
}
```

### Дебагер в різних IDE

| IDE | Команда |
|-----|---------|
| Visual Studio | F5 (start), F9 (breakpoint), F10/F11 (step) |
| VS Code | F5 (start), F9 (breakpoint), F10/F11 (step) |
| CLion | Shift+F9 (debug), Ctrl+F8 (breakpoint) |

### Правило: Використовуйте дебагер — він ефективніший за десятки `std::cout`.
