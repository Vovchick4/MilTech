# Урок №23. Багатофайлові програми

Реальні проекти рідко складаються з одного файлу. Розбиття на кілька файлів полегшує організацію, компіляцію та підтримку коду.

### Як це працює

Кожен `.cpp` файл компілюється окремо в об'єктний файл (`.o` або `.obj`). Потім лінкер з'єднує всі об'єктні файли в один виконуваний файл.

### Приклад багатофайлової програми

**add.cpp** — визначення функції:

```c++
int add(int a, int b)
{
    return a + b;
}
```

**main.cpp** — головна функція:

```c++
#include <iostream>

int add(int a, int b);  // прототип

int main()
{
    std::cout << add(3, 4);
    return 0;
}
```

Компіляція: `g++ -o program main.cpp add.cpp`

### Узгодженість типів

Прототип функції у файлі, де вона викликається, має точно збігатися з визначенням у файлі, де вона реалізована:

```c++
// main.cpp
int add(int a, int b);  // прототип

// add.cpp
double add(int a, int b) { return a + b; }  // ПОМИЛКА ЛІНКУВАННЯ! не збігається тип повернення
```

### Переваги багатофайлових програм

1. **Організація** — логічне групування коду (функції роботи з файлами, мережею тощо)
2. **Швидкість компіляції** — зміна одного файлу потребує перекомпіляції лише його
3. **Командна робота** — різні розробники працюють над різними файлами
4. **Повторне використання** — файл з корисними функціями можна використати в іншому проекті

### Правило: Розділяйте код на логічні модулі в окремих .cpp файлах.
