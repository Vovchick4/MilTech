# Урок №24. Заголовкові файли

`Заголовковий файл` (header file) — це файл з розширенням `.h` або `.hpp`, який містить прототипи функцій, визначення констант, оголошення класів тощо. Він дозволяє поширювати оголошення між кількома `.cpp` файлами.

### Навіщо вони потрібні?

Замість того, щоб писати прототип функції в кожному `.cpp` файлі, де вона використовується, ми пишемо його один раз у заголовковому файлі.

### Структура заголовкового файлу

**math_utils.h:**

```c++
#ifndef MATH_UTILS_H
#define MATH_UTILS_H

int add(int a, int b);
int multiply(int a, int b);

#endif
```

**math_utils.cpp:**

```c++
#include "math_utils.h"

int add(int a, int b)
{
    return a + b;
}

int multiply(int a, int b)
{
    return a * b;
}
```

**main.cpp:**

```c++
#include <iostream>
#include "math_utils.h"

int main()
{
    std::cout << add(3, 4);       // 7
    std::cout << multiply(5, 2);  // 10
    return 0;
}
```

### Лапки vs Кутові дужки

- `#include <iostream>` — кутові дужки для бібліотечних заголовків (шукаються в системних шляхах)
- `#include "math_utils.h"` — лапки для власних заголовків (шукаються в поточній директорії)

### Що можна розміщувати в заголовкових файлах?

- Прототипи функцій
- Визначення констант (`const`, `constexpr`)
- Оголошення класів, структур, enum
- Визначення inline-функцій

### Чого не можна розміщувати?

- Визначення звичайних функцій (крім inline) — призведе до помилки "multiple definition"
- Визначення змінних (крім constexpr) — призведе до дублювання

### Правило: Кожен .cpp файл повинен #include свій власний заголовковий файл.
