# Урок №18. Локальна область видимості

### Область видимості (scope)

`Область видимості` визначає, де в програмі доступна певна змінна. Локальні змінні оголошуються всередині функції або блоку `{}` і доступні лише там.

### Локальні змінні

```c++
#include <iostream>

void foo()
{
    int x { 5 };   // локальна змінна, доступна тільки в foo()
    std::cout << x;
}

int main()
{
    foo();
    // std::cout << x;  // ПОМИЛКА! x не видна тут
    return 0;
}
```

### Блокова область видимості

Змінна існує лише в межах свого блоку `{}`:

```c++
int main()
{
    int a { 1 };  // доступна у всьому main()
    
    {
        int b { 2 };  // доступна тільки в цьому блоці
        std::cout << a << b;  // OK
    }
    
    // std::cout << b;  // ПОМИЛКА! b вже не існує
    std::cout << a;  // OK
    return 0;
}
```

### Тривалість життя

Локальні змінні створюються при вході в блок і знищуються при виході з нього:

```c++
void example()
{
    int x { 5 };   // x створюється
    // робота з x
}                  // x знищується
```

### Затінення (shadowing)

Змінна внутрішнього блоку може "затінити" змінну зовнішнього блоку:

```c++
int main()
{
    int x { 10 };
    {
        int x { 20 };  // затінює зовнішню x
        std::cout << x;  // 20
    }
    std::cout << x;  // 10
    return 0;
}
```

### Правило: Уникайте затінення змінних — це призводить до плутанини.
