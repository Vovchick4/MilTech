# Урок №27. Конфлікт імен і простір імен std

### Конфлікт імен

`Конфлікт імен` виникає, коли два ідентифікатори мають однакове ім'я в одній області видимості:

```c++
#include <iostream>

int cout = 5;  // створюємо власну змінну cout

int main()
{
    std::cout << cout;  // компілятор плутає наш cout і std::cout
    return 0;
}
```

### Простір імен (namespace)

`Простір імен` (namespace) — це механізм для уникнення конфліктів імен. Він групує ідентифікатори в логічні групи.

```c++
namespace MyNamespace
{
    int cout = 5;
    void foo() { /* ... */ }
}

int main()
{
    std::cout << MyNamespace::cout;  // доступ через ::
    MyNamespace::foo();
    return 0;
}
```

### Простір імен std

`std` (standard) — простір імен Стандартної бібліотеки C++. Всі функції, об'єкти та типи бібліотеки знаходяться в ньому:

- `std::cout`, `std::cin`, `std::endl`
- `std::string`
- `std::vector`
- `std::sort()`

### Оператор ::

`Оператор розділення області видимості` `::` використовується для доступу до елементів простору імен:

```c++
std::cout << "Hello";  // cout з простору імен std
MyNamespace::foo();     // foo з нашого простору імен
```

### Проблема `using namespace std;`

```c++
using namespace std;  // погана практика!

int main()
{
    cout << "Hello";  //
}
```

Це "витягує" всі імена з `std` у глобальну область, що може призвести до конфліктів.

### Правило: Уникайте `using namespace std;`. Пишіть `std::` явно.
