# Урок №21. Базове форматування коду

Чистий, добре відформатований код легше читати, розуміти та підтримувати.

### Відступи

Кожен вкладений блок коду повинен мати відступ. Рекомендується 4 пробіли (або 1 табуляція):

```c++
int main()
{
    int x { 5 };          // відступ: 4 пробіли
    if (x > 0)
    {
        std::cout << x;   // відступ: 8 пробілів
    }
    return 0;
}
```

### Розташування фігурних дужок

Два поширені стилі:

**Стиль Java (K&R)** — відкриваюча дужка на тому ж рядку:

```c++
int main() {
    std::cout << "Hello";
    if (x > 0) {
        // код
    }
}
```

**Стиль Allman** — відкриваюча дужка на новому рядку:

```c++
int main()
{
    std::cout << "Hello";
    if (x > 0)
    {
        // код
    }
}
```

Обидва стилі прийнятні — оберіть один і дотримуйтеся його послідовно.

### Пробіли

Пробіли покращують читабельність, але не впливають на компіляцію:

```c++
// Погано
int x=5;
for(int i=0;i<10;i++){std::cout<<i;}

// Добре
int x = 5;
for (int i = 0; i < 10; i++) {
    std::cout << i;
}
```

### Довжина рядка

Намагайтеся обмежувати рядки 80-120 символами. Довгі рядки можна розбивати:

```c++
std::cout << "Це дуже довгий текст, який краще "
          << "розбити на кілька рядків для зручності."
          << std::endl;
```

### Однорідність

Найважливіше правило: будьте послідовними у форматуванні всього проекту.

### Правило: Форматуйте код так, щоб він був зрозумілим. Гарний код сам документує себе.
