# Урок №26. Header guards і директива #pragma once

### Проблема повторного включення

Якщо заголовковий файл включається кілька разів (наприклад, через ланцюжок `#include`), це призводить до помилки "multiple definition":

```c++
// school.h
#include "student.h"
#include "teacher.h"

// student.h (включає person.h)
#include "person.h"

// teacher.h (теж включає person.h)
#include "person.h"

// Коли school.h компілюється, person.h включається двічі!
```

### Header guards (традиційний спосіб)

`Header guard` — це набір директив препроцесора, які запобігають повторному включенню:

```c++
// person.h
#ifndef PERSON_H      // якщо PERSON_H не визначено
#define PERSON_H      // визначаємо PERSON_H

// вміст заголовкового файлу
int getAge();
std::string getName();

#endif                // кінець умовної компіляції
```

При першому включенні: `PERSON_H` не визначено → визначаємо → включаємо вміст.
При повторному включенні: `PERSON_H` вже визначено → пропускаємо вміст.

### #pragma once (сучасний спосіб)

`#pragma once` — простіша альтернатива:

```c++
// person.h
#pragma once

int getAge();
std::string getName();
```

Один рядок замість трьох. Більшість сучасних компіляторів підтримують `#pragma once`.

### Що вибрати?

| Header guards | #pragma once |
|--------------|--------------|
| Стандарт C++ | Розширення компілятора |
| Підтримується всіма компіляторами | Підтримується всіма сучасними компіляторами |
| Потрібен унікальний ідентифікатор | Покладається на шлях до файлу |
| Більше коду (3 рядки) | Один рядок |
| Захист навіть при однакових файлах | Може не спрацювати при symlinks |

### Правило: Використовуйте #pragma once — це простіше і менш схильне до помилок.
