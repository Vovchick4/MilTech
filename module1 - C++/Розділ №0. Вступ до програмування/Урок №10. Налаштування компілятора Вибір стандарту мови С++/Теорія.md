# Урок №10. Налаштування компілятора: Вибір стандарту мови С++

Як із величезної кількості різних версій C++ (C++98, C++03, C++11, C++14, C++17, C++20, C++23, C++26) компілятор розуміє, яку з них йому слід використовувати? Як правило, компілятор вибирає стандарт мови програмування за замовчуванням. Варто зауважити, що навіть у 2026 році стандарт за замовчуванням часто не є найновішим. Якщо ви хочете використовувати сучасні можливості мови, вам необхідно внести зміни в налаштування вашої IDE або компілятора. Ці налаштування застосовуються лише до поточного проєкту; при створенні нового проєкту процедуру доведеться повторити.

Зміст:

- Кодові імена версій мови С++
- Вибір стандарту мови С++ в Visual Studio
- Тестування вашого компілятора

# Кодові імена версій мови С++

Зверніть увагу, що стандарти мови С++ названі в честь тих років, в які вони були завершені/опубліковані (наприклад, C++17 був завершений та опублікований в 2017 році).

Однак, коли узгоджується новий стандарт мови програмування, то дуже часто буває незрозуміло, в якому році вдасться його завершити, тому поточному стандарту мови, який розробляється, дають кодове ім’я, яке потім замінюється на фактичну назву версії при публікуванні. Наприклад, C++11 мав кодове ім’я c++1x, коли над ним вели роботу. Ви і досі можете натикатися на подібні кодові імена версій С++, особливо, коли мова заходить про майбутню версію мови С++, у якої ще немає остаточної назви.

Кодові імена версій мови С++
Стандарти мови С++ названі на честь років, у які вони були офіційно опубліковані. Проте під час розробки нового стандарту йому дають тимчасове кодове ім’я. Ви все ще можете зустріти їх у документації або налаштуваннях компіляторів, особливо коли йдеться про майбутні версії.

Нижче подані кодові імена версій С++ та їхні фінальні назви:

c++1x = C++11

c++1y = C++14

c++1z = C++17

c++2a = C++20

c++2b = C++23

c++2c = C++26 (майбутній стандарт)

Наприклад, якщо ви бачите прапорець компілятора std:c++2b, це означає підтримку стандарту C++23.

### Вибір стандарту мови С++ в Visual Studio

Станом на 2026 рік, Visual Studio за замовчуванням може використовувати C++17 або C++20. Щоб розкрити потенціал нових стандартів (C++23 та експериментального C++26), необхідно підключити їх вручну для кожного проєкту.

Щоб змінити стандарт мови:

Відкрийте ваш проєкт.

Клацніть правою кнопкою миші по назві проєкту в "Оглядач рішень" (Solution Explorer) > "Властивості" (Properties).

У вікні властивостей переконайтеся, що в пункті "Конфігурація" (Configuration) вибрано "Всі конфігурації" (All Configurations).

Перейдіть на вкладку "C/C++" > "Мова" (Language).

У пункті "Стандарт мови С++" (C++ Language Standard) виберіть потрібну версію.

Рекомендація на 2026 рік:
Для стабільної роботи більшості сучасних бібліотек (наприклад, для MilTech чи AI-застосунків) обирайте "Стандарт ISO C++23 (/std:c++23)".

Якщо ви хочете випробувати найновіші фішки, які щойно з'явилися в індустрії, обирайте пункт "Попередня версія з найновішого робочого набору С++ (/std:c++latest)", що відповідає напрацюванням C++26.

На момент написання даної статті, я рекомендую вибрати "Стандарт ISO C++17 (/std:c++17)", який є найновішим стабільним стандартом, який підтримує Visual Studio.

Якщо ви хочете поекспериментувати з можливостями майбутнього стандарту мови C++20, то ви можете вибрати пункт "Попередня версія ... (/std:c++latest)". Просто пам’ятайте, що його підтримка може мати баги.

### Тестування вашого компілятора

Після налаштування C++23 або вище, ви можете перевірити роботу компілятора. Наступна програма використовує можливості стандартів C++20/23 (наприклад, консепту та std::expected або покращені views) і повинна скомпілюватися без помилок:

```cpp
#include <iostream>
#include <vector>
#include <ranges>
#include <concepts>

// Використання концептів (C++20)
template <typename T>
concept Numeric = std::is_arithmetic_v<T>;

int main()
{
    std::vector<int> numbers = { 1, 2, 3, 4, 5, 6 };

    // Використання Ranges (C++20) та Views
    auto result = numbers | std::views::filter([](int n) { return n % 2 == 0; })
                          | std::views::transform([](int n) { return n * n; });

    std::cout << "Square of even numbers: ";
    for (int n : result) {
        std::cout << n << " ";
    }
    std::cout << '\n';

    // Перевірка версії стандарту через макрос
    std::cout << "C++ version: " << __cplusplus << '\n';

    return 0;
}
```

Якщо код не компілюється, перевірте, чи встановлені у вас останні оновлення для Visual Studio 2022 або Visual Studio 2025. У 2026 році підтримка C++23 є повною у більшості актуальних компіляторів.
