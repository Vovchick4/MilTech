# 📘 Урок №3. Введення в розробку програмного забезпечення на C++

Перед написанням першої програми важливо розуміти життєвий цикл розробки ПЗ. Нижче наведено покроковий гайд: від ідеї до готового файлу.



## 🛠 Процес розробки (Step-by-Step)

### 1️⃣ Крок №1: Визначте проблему ("Що?")
Перший крок — чітке формулювання ідеї. Ви повинні розуміти, що саме має робити програма.
*   **Приклад:** *"Програма для розрахунку середнього арифметичного"* або *"2D-лабіринт для користувача"*.

### 2️⃣ Крок №2: Визначте рішення ("Як?")
Нехтування цим кроком — головна помилка новачків. Добре рішення має бути:
*   **Простим** (без зайвих ускладнень).
*   **Документованим** (з коментарями).
*   **Модульним** (частини коду можна перевикористати).
*   **Надійним** (обробка помилок).

> 💡 **Закон Парето в програмуванні:** 20% часу витрачається на написання коду, а 80% — на його відлагодження (debug) та підтримку.

---

### 3️⃣ Крок №3: Напишіть програму
Для написання коду використовуйте **IDE** (середовище розробки). Основні вимоги до редактора:
1. **Нумерація рядків** (легше шукати помилки за номером).
2. **Підсвічування синтаксису** (краще сприйняття структур).
3. **Моноширинний шрифт** (щоб не плутати `0` та `O` або `1` та `l`).

| Тип файлу | Розширення | Опис |
| :--- | :--- | :--- |
| **Вихідний код** | `.cpp` | Містить ваші інструкції мовою C++ |
| **Об'єктний файл** | `.o` / `.obj` | Результат компіляції (машинний код) |
| **Виконуваний файл** | `.exe` / `bin` | Готова програма для запуску |

---

### 4️⃣ Крок №4: Компіляція програми
Компілятор виконує дві функції:
1. Перевіряє **синтаксис** (чи немає помилок у правилах мови).
2. Створює **об'єктний файл** (конвертує код у машинну мову).



**Приклад команди для Linux/macOS (g++):**
```bash
# Компіляція трьох файлів у об'єктні модулі (.o)
g++ -c file1.cpp file2.cpp file3.cpp
```

---

### 5️⃣ Крок №5: Лінкінг (З'єднання)
Лінкер (linker) збирає всі ваші `.o` файли та функції зі **Стандартної бібліотеки C++** в один файл.



**Команди для збірки:**
```bash
# Окремий лінкінг вже скомпільованих файлів
g++ -o my_program file1.o file2.o file3.o

# Компіляція + Лінкінг одним кроком (найчастіший варіант)
g++ -o my_program file1.cpp file2.cpp file3.cpp
```
