# Урок №1. Основи Git

Git — система контролю версій для відстеження змін у коді.

## Ініціалізація

```bash
# Створити новий репозиторій
git init

# Клонувати існуючий
git clone https://github.com/user/repo.git

# Перевірити стан
git status
```

## Основні команди

```bash
# Додати файли в індекс
git add file.txt       # конкретний файл
git add .              # всі зміни
git add *.cpp          # всі .cpp файли

# Зберегти зміни (коміт)
git commit -m "Опис змін"

# Переглянути історію
git log
git log --oneline      # скорочений вигляд
git log --graph        # граф гілок
```

## Життєвий цикл файлу

```
Working Directory → git add → Staging Area → git commit → Repository
     ↑                                        │
     └────────── git checkout ────────────────┘
```

## Налаштування

```bash
git config --global user.name "Ім'я"
git config --global user.email "email@example.com"
git config --global core.editor "code --wait"
```

## .gitignore

```gitignore
# Ігнорувати файли
*.exe
*.obj
*.log
build/
.vscode/
__pycache__/
.DS_Store
```

## Приклад роботи

```bash
mkdir myproject
cd myproject
git init

echo "# My Project" > README.md
git add README.md
git commit -m "Initial commit: README"

echo '#include <iostream>
int main() { std::cout << "Hello"; }' > main.cpp
git add main.cpp
git commit -m "Add main.cpp"
```

## diff — перегляд змін

```bash
git diff              # непроіндексовані зміни
git diff --staged     # проіндексовані
git diff HEAD         # всі зміни
```

## Практичні поради

1. Комітити часто, малими порціями
2. Писати зрозумілі повідомлення
3. Не комітити згенеровані файли
4. Перевіряти git status перед комітом
