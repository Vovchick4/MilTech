# Урок №5. Командна робота

## Fork + Pull Request (Open Source)

```
1. Fork репозиторію на GitHub
2. git clone вашого fork
3. Створити гілку: git checkout -b feature
4. Зробити зміни, коміти
5. git push origin feature
6. Відкрити Pull Request
```

## Code Review

```bash
# Перед PR: оновити свою гілку
git checkout main
git pull origin main
git checkout feature
git merge main
# або:
git rebase main
```

## Робота в команді

```bash
# Додати upstream (оригінальний репозиторій)
git remote add upstream https://github.com/original/repo.git

# Отримати зміни з upstream
git fetch upstream
git checkout main
git merge upstream/main

# Відправити в свій fork
git push origin main
```

## Вирішення конфліктів

```cpp
// Конфлікт виникає, коли дві людини змінили той самий рядок
// 1. Відкрити файл з конфліктом
// 2. Знайти маркери:

<<<<<<< HEAD
int maxValue = 100;
=======
int maxValue = 200;
>>>>>>> feature

// 3. Вибрати потрібне або об'єднати:
int maxValue = 200;

// 4. Зберегти, додати, комітнути
git add main.cpp
git commit -m "Resolve conflict in maxValue"
```

## Запобігання конфліктам

1. Розбивати код на модулі
2. Комунікувати з командою
3. Часто робити pull
4. Використовувати .gitattributes

## Protected branches

Налаштування в GitHub/GitLab:

- Захист main від прямого push
- Вимога Code Review
- Автоматичні перевірки (CI)
- Мінімум 1 схвалення

## Git Flow

```
main ─────●──────────●──────────────●
           \        / \            /
develop     ●──────●   ●──────────●
              \   /     \   \   /
features      ●─●       ●───●
```

## Правило

Перед PR завжди оновлюйте свою гілку від main. Пишіть описові PR.
