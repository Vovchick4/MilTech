# Урок №6. Розширені можливості Git

## Rebase

Rebase — перенесення комітів на нову основу:

```bash
# Замість merge
git checkout feature
git rebase main
```

```
До rebase:
main:     A---B---C
feature:      D---E

Після rebase:
main:     A---B---C
feature:            D'---E'
```

## Rebase vs Merge

```bash
# Merge — створює коміт злиття
git checkout main; git merge feature

# Rebase — лінійна історія (без merge коміту)
git checkout feature; git rebase main; git checkout main; git merge feature
```

## Interactive rebase

```bash
# Змінити останні 3 коміти
git rebase -i HEAD~3

# Команди:
# pick — залишити коміт
# reword — змінити повідомлення
# edit — змінити коміт
# squash — об'єднати з попереднім
# fixup — об'єднати, відкинувши повідомлення
# drop — видалити коміт
```

## Cherry-pick

Застосувати конкретний коміт в поточну гілку:

```bash
git cherry-pick abc123   # застосувати коміт abc123
git cherry-pick abc123..def456  # діапазон
```

## Bisect — пошук помилки

```bash
# Почати пошук
git bisect start
git bisect bad HEAD    # поточна версія погана
git bisect good v1.0   # версія v1.0 хороша

# Git перебирає коміти, ви позначаєте:
git bisect good  # якщо помилки немає
git bisect bad   # якщо помилка є

# Закінчити
git bisect reset
```

## Submodules

```bash
# Додати підмодуль
git submodule add https://github.com/user/lib.git libs/lib

# Клонувати з підмодулями
git clone --recurse-submodules https://github.com/user/project.git

# Оновити підмодулі
git submodule update --init --recursive
```

## Hooks (гачки)

```bash
# .git/hooks/pre-commit — виконується перед комітом
#!/bin/sh
echo "Running tests..."
make test
if [ $? -ne 0 ]; then
    echo "Tests failed!"
    exit 1
fi
```

## Аліаси

```bash
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status
git config --global alias.lg "log --oneline --graph --all"
```

## Інструменти

```bash
# Зберегти пароль
git config --global credential.helper store

# Перенести гілку
git rebase --onto target start end
```

## Правило

Rebase для локальних гілок, merge — для публічних. Не робіть rebase на спільних гілках.
