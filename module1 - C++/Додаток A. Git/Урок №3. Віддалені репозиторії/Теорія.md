# Урок №3. Віддалені репозиторії

Віддалені репозиторії (remote) — версія проекту на іншому сервері (GitHub, GitLab).

## Додавання remote

```bash
# Переглянути віддалені репозиторії
git remote -v

# Додати remote
git remote add origin https://github.com/user/repo.git

# Змінити URL
git remote set-url origin https://github.com/user/repo.git
```

## push — відправити зміни

```bash
# Відправити в remote
git push origin main

# Відправити всі гілки
git push --all origin

# Встановити зв'язок (upstream)
git push -u origin feature-branch

# Видалити гілку з remote
git push origin --delete feature-branch
```

## pull — отримати зміни

```bash
# Отримати і злити
git pull origin main

# Скорочено (якщо upstream налаштовано)
git pull

# Тільки отримати (без merge)
git fetch origin
```

## fetch vs pull

```
git fetch → оновлює remote/gілки, не змінює робочу директорію
git pull  → git fetch + git merge (автоматичне злиття)
```

## Робота з GitHub

```bash
# Клонувати репозиторій
git clone https://github.com/user/repo.git

# Створити fork → зробити зміни → Pull Request

# Відправити зміни у свій fork
git push origin feature

# Pull Request — запит на злиття в оригінальний репозиторій
```

## Приклад: перший push

```bash
git init
git add .
git commit -m "First commit"
git remote add origin https://github.com/user/repo.git
git branch -M main
git push -u origin main
```

## Приклад: оновлення

```bash
git pull origin main           # отримати нове
# ... зробити зміни ...
git add .
git commit -m "New feature"
git push origin main           # відправити
```

## Відстеження гілок

```bash
# Створити локальну гілку з віддаленої
git checkout -b feature origin/feature

# Подивитися віддалені гілки
git branch -r

# Подивитися всі гілки (локальні + віддалені)
git branch -a
```

## Правило

Перед початком роботи робіть `git pull`, перед завершенням — `git push`.
