# Урок №2. Гілки в Git

Гілки (branches) дозволяють вести паралельну розробку.

## Основні операції

```bash
# Список гілок
git branch

# Створити гілку
git branch feature-login

# Переключитися на гілку
git checkout feature-login
# або (сучасний синтаксис)
git switch feature-login

# Створити і переключитися
git checkout -b feature-login
git switch -c feature-login
```

## Злиття (merge)

```bash
# Злити гілку в поточну
git checkout main
git merge feature-login

# Видалити гілку
git branch -d feature-login      # локально
git push origin --delete feature-login  # віддалено
```

## Візуалізація

```
main:     A---B---C---D---E
                \
feature:         C---D
```

## Конфлікти при merge

```bash
# Якщо merge викликав конфлікт:
# 1. Відкрити файли з конфліктами
# 2. Вибрати потрібний код
# 3. Зберегти і додати:
git add .
git commit -m "Merge: resolve conflicts"
```

Приклад конфлікту у файлі:

```cpp
<<<<<<< HEAD
int value = 10;
=======
int value = 20;
>>>>>>> feature
```

## Стратегії роботи з гілками

```
main ───●────────●────────●────────●
         \      / \      / \      /
dev        ●──●   ●──●   ●──●
              \   /   \   /
features       ●─●     ●─●
```

## Найпоширеніші назви гілок

- `main` / `master` — основна стабільна версія
- `develop` / `dev` — розробка
- `feature/назва` — нова функція
- `bugfix/назва` — виправлення помилки
- `release/версія` — підготовка релізу

## Утиліти

```bash
# Перейменувати гілку
git branch -m old-name new-name

# Подивитися останній коміт на кожній гілці
git branch -v

# Змерджити без fast-forward (явний merge commit)
git merge --no-ff feature
```

## Правило

Тримайте main завжди стабільним. Нові функції розробляйте в окремих гілках.
