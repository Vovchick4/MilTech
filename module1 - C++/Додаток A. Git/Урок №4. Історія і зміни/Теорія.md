# Урок №4. Історія і зміни

Як переглядати історію, скасовувати зміни та аналізувати коміти.

## Перегляд історії

```bash
# Звичайний лог
git log

# Скорочений
git log --oneline

# Граф
git log --graph --oneline --all

# З датами
git log --oneline --since="2024-01-01"

# По автору
git log --author="Ім'я"

# По файлу
git log --oneline main.cpp
```

## Подивитися коміт

```bash
git show HEAD        # останній коміт
git show abc123      # конкретний коміт (хеш)
git show HEAD~1      # передостанній
git show HEAD~3      # третій з кінця
```

## Скасування змін

```bash
# Скасувати зміни в файлі (до останнього коміту)
git checkout -- file.txt

# Забрати файл з індексу
git reset HEAD file.txt

# Скасувати коміт (новий коміт, що скасовує)
git revert HEAD

# Повністю видалити коміт
git reset --hard HEAD~1

# М'який reset (файли залишаються)
git reset --soft HEAD~1
```

## Види reset

```
--soft:   HEAD ← старий коміт, індекс і файли не змінюються
--mixed:  HEAD ← старий коміт, індекс змінюється, файли ні
--hard:   HEAD ← старий коміт, все змінюється (небезпечно!)
```

## stash — тимчасове збереження

```bash
# Зберегти незавершені зміни
git stash

# Зберегти з описом
git stash push -m "WIP: login feature"

# Список
git stash list

# Застосувати останній
git stash pop

# Застосувати конкретний
git stash apply stash@{1}

# Видалити
git stash drop stash@{0}
```

## tag — мітки

```bash
# Створити тег
git tag v1.0.0
git tag -a v1.0.0 -m "Version 1.0.0"

# Список тегів
git tag

# Відправити тег в remote
git push origin v1.0.0

# Переключитися на тег
git checkout v1.0.0
```

## Приклад: знайти помилку

```bash
# Хто писав цей код?
git blame main.cpp

# Коли змінили?
git log -p -- main.cpp

# Де шукати?
git log --oneline -S "buggyFunction"
```

## Правило

Використовуйте `git revert` для публічних комітів, `git reset` — тільки для локальних.
